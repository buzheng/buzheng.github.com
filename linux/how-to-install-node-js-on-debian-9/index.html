如何在 Debian 9 上安装 Node.js 和 npm

2019年3月27日 • 约 1100 字 , 预计阅读时间 5 分钟

Node.js 是一个开源的跨平台 JavaScript 运行时环境，构建于 Chrome 的 JavaScript 引擎之上，提供了在服务器端执行 JavaScript 代码的能力。它主要用于构建服务器端应用程序，但它作为全栈和前端解决方案也非常受欢迎。 npm 是 Node.js 的默认包管理器，也是世界上最大的软件注册仓库。

在本教程中，我们将介绍如何在 Debian 9 上安装 Node.js 和 npm 。我们将向您展示在 Debian 上安装 Node.js 的两种不同方法。您只需要选择最适合您的安装方法。

从 NodeSource 软件仓库安装 Node.js 和 NPM

安装最新版本的 Node.js 和 npm 的最简单方法是从 NodeSource 维护的软件仓库安装软件包。

要将 NodeSource 软件仓库添加到系统，请运行以下命令：

curl -sL https://deb.nodesource.com/setup_8.x | sudo bash -

Node.js 的当前 LTS 版本是版本 8.x ， Carbon 。如果你想安装 Node.js 的 10.x 版本，只需要把 setup_8.x 改为 setup_10.x

添加 NodeSource 软件仓库后，使用以下命令安装 Node.js 和 npm ：

sudo apt install nodejs

要验证安装是否成功，请打印 Node.js 和 npm 版本：

node --version

v8.11.3

npm --version

5.6.0

使用 NVM 安装 Node.js 和 NPM

NVM （Node 版本管理器）是一个 bash 脚本，允许您管理多个 Node.js 版本。使用 NVM ，您可以安装和卸载要使用或测试的任何特定 Node.js 版本。

使用以下 curl 命令下载 nvm 安装脚本：

curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.33.11/install.sh | bash

安装脚本从 Github 软件仓库克隆 nvm 到 ~/.nvm 目录，并将 nvm 路径添加到 Bash 或 ZSH 配置文件中。

=> Close and reopen your terminal to start using nvm or run the following to use it now:

export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"  # This loads nvm
[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"  # This loads nvm bash_completion

如上面的输出所示，您可以打开新的 shell 会话或运行命令将 nvm 脚本的路径添加到当前会话。任选一种就行。

要确保正确安装 nvm ，请输入：

nvm --version

0.33.11

既然已经在 Debian 机器上安装了 nvm，要安装最新的 Node.js 版本，请输入：

nvm install node

Downloading and installing node v10.5.0...
Downloading https://nodejs.org/dist/v10.5.0/node-v10.5.0-linux-x64.tar.xz...
######################################################################## 100.0%
Computing checksum with sha256sum
Checksums matched!
Now using node v10.5.0 (npm v6.1.0)
Creating default alias: default -> node (-> v10.5.0)

输入以下命令验证 Node.js 版本：

node --version

v10.5.0

让我们再安装两个版本，最新的 LTS 版本和 4.9.1 版本

nvm install --lts
nvm install 6.14.3

安装 LTS 版本和 6.14.3 后，我们可以通过输入以下内容列出已安装的 Node.js 实例：

nvm ls

->      v6.14.3                     # ACTIVE VERSION
        v8.11.3
        v10.5.0
         system
default -> node (-> v10.5.0)        # DEFAULT VERSION
node -> stable (-> v10.5.0) (default)
stable -> 10.5 (-> v10.5.0) (default)
iojs -> N/A (default)
lts/* -> lts/carbon (-> v8.11.3)
lts/argon -> v4.9.1 (-> N/A)
lts/boron -> v6.14.3
lts/carbon -> v8.11.3

在上面的输出中，右侧带箭头的条目 (-> v6.14.3) 是当前 shell 会话中使用的版本，默认版本设置为 v10.5.0 。

打开新 shell 会话时将使用默认版本。

如果要更改当前活动的版本，请使用以下命令：

nvm use 8.11.3

并通过输入以下来验证它：

nvm current

v8.11.3

要将版本 8.11.3 设置为默认的 Node.js 版本，请执行：

nvm alias default 8.11.3

安装开发工具

要从 npm 软件仓库编译和安装本机加载项，您需要安装开发工具。

sudo apt install build-essential

卸载 Node.js.

如果由于某些原因要卸载 Node.js 和 npm 软件包，可以使用以下命令：

sudo apt remove nodejs npm

结论

我们向您展示了在 Debian 9 服务器上安装 Node.js 和 npm 的两种不同方法。您选择的方法取决于您的要求和偏好。尽管从 NodeSource 软件仓库安装打包版本更容易，但 nvm 方法为您提供了更大的灵活性，可以在每个用户的基础上添加和删除不同的 Node.js 版本。

现在您已经在 Debian 9 系统上安装了 Node.js ，是时候开始开发您的应用程序了！

如果你想用 yarn 管理你的 npm 包，你可以查看本教程，了解如何在 Debian 9 上安装和使用 yarn。
