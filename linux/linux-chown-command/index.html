Linux Chown命令

2019年5月9日 • 约 1000 字 , 预计阅读时间 4 分钟

在Linux中，所有文件都与所有者和组相关联。chown命令用于更改给定文件，目录或链接的用户和组所有权。

在本教程中，我们将通过简单的示例向您展示如何使用chown命令。

Chown命令语法

在讨论如何使用chown命令之前，让我们先回顾一下基本语法。

chown命令表达式采用以下形式：

chown [OPTIONS] USER[:GROUP] FILE(s)

USER是新所有者的用户名或用户ID（UID），GROUP是新组的名称或组ID（GID），FILE(s)是一个或多个文件，目录或链接的名称。

使用该ls -l命令可以找出谁拥有文件或属于哪个组。

为了能够更改文件的所有权，运行chown命令的用户必须具有sudo权限。

如何更改文件的所有者

要更改文件的所有者，请使用chown命令，后跟新所有者的用户名和目标文件。

chown USER FILE

例如，以下命令将更改名为file1新名为的所有者的文件的所有权linuxize：

chown linuxize file1

要更改多个文件或目录的所有权，请将它们指定为以空格分隔的列表。例如，下面的命令将名为file1directory的目录的所有权更改为名为dir1的新所有者linuxize：

chown linuxize file1 dir1

可以使用数字用户ID（UID）代替用户名。以下示例将更改名为file2UID为1000的新所有者的文件的所有权：

chown 1000 file2

如果数字所有者作为用户名存在，则所有权将转移到用户名。

如何更改文件的所有者和组

要更改文件的所有者和组，请使用chown命令后跟由冒号（:）分隔的新所有者和组，而不使用插入空格和目标文件。

chown USER:GROUP FILE

以下命令将更改名为name file1new linuxize和group 的新所有者的文件的所有权users：

chown linuxize:users file1

如果在冒号（:）后省略组名，则将文件组更改为指定用户的登录组。

chown linuxize: file1

如何更改文件组

该chown命令可以执行与命令相同的功能chgrp，即它可以更改文件组。

要仅更改文件组，请使用chown命令后跟冒号（:）和新组名称以及目标文件。

chown :GROUP FILE

以下命令将更改名为文件的所属组file1到www-data：

chown :www-data file1

如何递归更改文件所有权

要递归操作输入目录下的所有文件和目录，请使用-r（--recursive）选项：

chown -R USER:GROUP DIRECTORY

例如，以下命令会将目录下所有文件和目录的所有权更改/var/www为名为的新所有者和组www-data：

chown www-data: /var/www

结论

到目前为止，您应该很好地理解如何使用Linux chown命令。如果您想了解有关该chown命令的更多信息，请访问chown手册页。
