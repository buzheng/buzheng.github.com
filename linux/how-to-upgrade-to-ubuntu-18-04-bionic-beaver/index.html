如何将 Ubuntu 升级到 18.04 (Bionic Beaver)

2019年3月12日 • 约 1200 字 , 预计阅读时间 5 分钟

最新的 Ubuntu LTS 版本 Ubuntu 18.04 (Bionic Beaver) 于 2018 年 4 月 26 日发布，它会被支持 5 年，一直直到 2023 年 4 月。在本教程中，我们将向您展示如何从 Ubuntu 16.04 或更高版本升级到 Ubuntu 18.04 (Bionic Beaver) 。

先决条件

在开始本教程之前，请确保以具有 sudo 权限的用户身份登录。

备份您的数据

首先，也是最重要的一点，请一定在开始升级操作系统之前备份数据。如果您在虚拟机上运行 Ubuntu ，最好备份完整的系统快照，以便在更新失败的时候轻松恢复您的计算机。

更新当前安装的软件包

在开始版本升级之前，建议将所有当前安装的软件包更新到其最新版本。

sudo apt update
sudo apt upgrade

我们还将运行 dist-upgrade，它会执行已安装软件包的主要版本升级，并可能删除一些不必要的软件包：

sudo apt dist-upgrade

将 Ubuntu 16.04 升级到 Ubuntu 18.04

Ubuntu 提供了一个工具 do-release-upgrade ，它让升级过程非常简单。该工具应该已经安装在您的系统上，但如果由于某种原因没有安装它，您可以安装它：

sudo apt install do-release-upgrade

运行 do-release-upgrade, 开始升级：

sudo do-release-upgrade

如果您像我一样通过 SSH 升级系统，系统会询问您是否要继续。

在升级过程中，您将被问到各种问题，例如您是要保留现有配置文件还是安装软件包维护者的版本。如果您没有对文件进行任何自定义更改，则应键入Y；如果要保持当前配置则输入N。

升级过程可能需要一些时间，具体取决于更新次数和网络的下载速度。

安装新软件包后，更新工具会询问您是否要删除过时的软件，如果您不确定键入 d 查看过时软件包列表，在大多数情况下，输入 y 并删除所有过时的软件包是安全的：

升级过程完成并假设一切顺利后，系统会要求您重新启动计算机。输入 y 以继续：

Restart required

确认更新

等待服务器重启完成，然后重新连接。登录后，您将收到以下消息：

Welcome to Ubuntu 18.04 LTS (GNU/Linux 4.15.0-20-generic x86_64)

您还可以使用以下命令检查 Ubuntu 版本：

lsb_release -a
No LSB modules are available.
Distributor ID: Ubuntu
Description:    Ubuntu 18.04 LTS
Release:    18.04
Codename:   bionic

到现在为止，你现在可以享用新的 Ubuntu 18.04 LTS (Bionic Beaver) 了。
