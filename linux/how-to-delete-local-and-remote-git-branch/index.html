如何删除本地和远程Git分支

May 9, 2019

分支是日常开发过程的一部分，也是Git中最强大的功能之一。一旦分支合并，除了历史研究之外，它没有任何意义。成功合并后删除分支是常见的建议做法。

本指南介绍了如何删除本地和远程Git分支。

删除本地Git分支

要删除本地Git分支，请使用git branch带-d（--delete）标志的命令：

git branch -d branch_name
Deleted branch branch_name (was 17d9aa0).

如果您尝试删除包含尚未合并的更改的分支，您将收到以下错误：

error: The branch 'branch_name' is not fully merged.
If you are sure you want to delete it, run 'git branch -D branch_name'.

如上面的消息所示，您可以使用-D标志强制删除，该标志是以下的快捷方式--delete --force：

git branch -D branch_name

请注意，如果删除未合并的分支，您将丢失该分支上的所有更改。

要列出包含未合并更改的所有分支，您可以使用git branch --no-merged。

如果您已检入要删除的分支，则会收到以下消息：

error: Cannot delete branch 'branch_name' checked out at '/path/to/repository'

您无法删除活动分支。首先，切换到另一个分支，然后删除branch_name：

git checkout mastergit branch -d branch_name

删除远程Git分支

在Git中，本地和远程分支是单独的对象。删除本地分支不会删除远程分支。

要删除远程分支，请使用git push带-d（--delete）标志的命令：

git push remote_name --delete branch_name

remote_name通常在哪里origin：

git push origin --delete branch_name
...
 - [deleted]         branch_name

还有一个删除远程分支的替代命令，也就是说，至少对我来说难以记住：

git push origin remote_name :branch_name

如果您正在与一组人一起处理项目并尝试删除已被其他人删除的远程分支，您将收到以下错误消息：

error: unable to push to unqualified destination: branch_name The destination refspec neither matches an existing ref on the remote nor begins with refs/, and we are unable to guess a prefix based on the source ref. error: failed to push some refs to 'git@example.com:/my_repo'

在这种情况下，您需要将分支列表与以下内容同步：

git fetch -p

-p flag将告诉Git在获取之前删除远程存储库中不再存在的任何远程跟踪引用。

结论

在本教程中，您学习了如何删除本地和远程Git分支。分支基本上是对更改快照的引用，生命周期较短。一旦分支合并到主（或另一个主分支），就不再需要它，应该将其删除。
