如何在 Ubuntu 18.04 上安装 Git

March 30, 2019

Git 是分布式版本控制系统的事实标准，现在被大多数开发人员使用。它允许您跟踪代码更改，恢复到以前的阶段，创建分支以及与您的开发人员协作。

Git 最初是由 Linux 内核的创建者 Linus Torvalds 开发的。

本教程将指导您完成在 Ubuntu 18.04 上安装 Git 所需的步骤。相同的说明适用于 Ubuntu 16.04 和任何其他基于 Ubuntu 的发行版，包括 Kubuntu ， Linux Mint 和 Elementary OS 。

先决条件

在继续本教程之前，请确保以具有 sudo 权限的用户身份登录。

用 apt 安装 Git

安装 Git 的最简单和推荐的方法是使用 Ubuntu 默认存储库中的 apt 包管理工具安装它。如果要从源代码安装最新的稳定版 Git ，请转到本教程的 “从源代码安装 Git” 部分。

按照以下步骤在您的 Ubuntu 系统上安装 Git ：

  1. 首先更新包索引：

    sudo apt update
    1. 运行以下命令安装 Git ：
    sudo apt install git

  2. 通过输入以下命令验证安装，该命令将打印 Git 版本：

    git --version
    git version 2.17.1

在撰写本文时， Ubuntu 18.04 存储库中提供的当前 Git 版本是 2.17.1 ，它只是比最新 Git 版本(2.18.0) 上一个版本而已。

从源代码安装 Git

另一种选择是从源代码编译 Git ，这将允许您安装最新的 Git 版本并自定义构建选项，但您无法通过 apt 软件包管理器维护 Git 安装。

首先，您需要在 Ubuntu 系统上安装构建 Git 所需的依赖项：

sudo apt updatesudo apt install make libssl-dev libghc-zlib-dev libcurl4-gnutls-dev libexpat1-dev gettext unzip

安装完成后，打开浏览器，访问 GitHub 上的 Git 项目镜像，并复制以 .tar.gz 结尾的最新版本链接 URL ：

在撰写本文时，最新的稳定 Git 版本是 2.18.0 。

我们将在 /usr/src 目录中下载 Git 源代码，该目录是放置源文件的常用位置，跳转到目录 /usr/src ：

cd /usr/src/

使用 wget 命令下载 tar.gz 文件并保存为 git.tar.gz ：

sudo wget https://github.com/git/git/archive/v2.18.0.tar.gz -O git.tar.gz

接下来，通过输入以下内容来解压缩 tarball 并切换到 git 源目录：

sudo tar -xf git.tar.gz
cd git-*

运行以下两个命令在 Ubuntu 系统上编译和安装 Git ：

sudo make prefix=/usr/local all
sudo make prefix=/usr/local install

输入以下命令验证安装，该命令将打印已安装的 Git 版本：

git --version
git version 2.18.0

如果要升级到较新版本，则需要重复安装过程。

配置 git

现在您已经安装了 git ，需要设置以下在提交代码时使用的个人信息：

git config --global user.name "Your Name"git config --global user.email "youremail@yourdomain.com"

要验证配置更改，可以使用以下命令：

git config --list
user.name=Your Name
user.email=youremail@yourdomain.com

配置设置存储在 ~/.gitconfig 文件中：

[user]
    name = Your Name
    email = youremail@yourdomain.com

如果要对 Git 配置进行进一步更改，可以使用 git config 命令或手动编辑 ~/.gitconfig 文件。

结论

您已经学习了如何在 Ubuntu 系统上安装 Git 。您现在应该查看 Pro Git 书籍并了解有关如何使用 Git 的更多信息。
