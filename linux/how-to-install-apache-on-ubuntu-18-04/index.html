如何在 Ubuntu 18.04 上安装 Apache

2019年3月25日 • 约 1000 字 , 预计阅读时间 4 分钟

Apache HTTP 服务器是世界上最受欢迎的 Web 服务器。它是一个免费的，开源的跨平台 HTTP 服务器，提供强大的功能，还可以通过各种模块进行扩展。

本教程介绍如何在 Ubuntu 18.04 上安装和管理 Apache Web 服务器。

先决条件

在开始本教程之前，请确保以具有 sudo 权限的用户身份登录。

安装 Apache

Apache 在默认的 Ubuntu 存储库中可用，因此我们可以很轻松的使用 apt 包管理工具安装它。在 Ubuntu 和 Debian 系统上，Apache 包和服务的名称是 apache2 。

首先，更新软件包索引，然后使用以下命令安装 apache2 软件包：

sudo apt update
sudo apt install apache2

就是这样， Apache 已经安装并自动启动。您可以通过输入以下命令来检查 Apache 服务状态：

sudo systemctl status apache2

调整防火墙

如果您的 Ubuntu 服务器受防火墙保护，则需要打开 HTTP (80) 和 HTTPS (443) 端口。

假设您正在使用 UFW 管理防火墙，可以使用以下命令打开必要的端口：

sudo ufw allow 'Apache Full'

您可以使用以下方式验证更改：

sudo ufw status

Status: active

Apache 配置文件的结构和最佳实践

  • 所有 Apache 配置文件都位于 /etc/apache2 目录中。
  • 主要的 Apache 配置文件是 /etc/apache2/apache2.conf 。
  • Apache 将侦听的端口在 /etc/apache2/ports.conf 中指定。
  • Apache 虚拟主机文件存储在 /etc/apache2/sites-available 目录中。除非链接到 /etc/apache2/sites-enabled 目录，否则 Apache 不使用此目录中的配置文件。
  • 要激活虚拟主机，您需要使用 a2ensite 命令在目录 sites-enabled 中创建符号链接指向 sites-availables 目录中的配置文件。要停用虚拟主机，请使用 a2dissite 该命令。
  • 遵循标准命名约定是个好主意。例如，如果您的域名是 mydomain.com 那么虚拟主机配置文件应该命名为 /etc/apache2/sites-available/mydomain.com.conf
  • 负责加载各种 Apache 模块的配置文件位于 /etc/apache2/mods-available/ 目录中。 mods-available 目录中的配置文件可以使用 a2enconf 命令通过在 /etc/apache2/mods-enable/ 目录创建指向此目录的符号链接来激活。使用 a2disconf 命令来禁用目录中的配置 。
  • 包含全局配置片段的文件存储在 /etc/apache2/conf-available/ 目录中。 conf-available中的文件可以通过使用 a2enconf 命令在 /etc/apache2/conf-enabled/ 创建符号链接来启用。 使用命令 a2disconf 禁用。
  • Apache 日志文件 (access.log 和 error.log) 位于 /var/log/apache/ 目录中。建议每个虚拟主机都有不同 access 和 error 日志文件。
  • 您可以将网站文档根目录设置为所需的任何位置。 webroot 最常见的位置包括：
    • /home/<user_name>/<site_name>
    • /var/www/<site_name>
    • /var/www/html/<site_name>
    • /opt/<site_name>

结论

您已在 Ubuntu 18.04 服务器上成功安装了 Apache 。您现在已经可以开始部署您的应用程序并将 Apache 用作 Web 或代理服务器了。
