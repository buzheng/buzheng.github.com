如何在 Ubuntu 18.04 上安装 MongoDB

April 26, 2019

MongoDB 是一个免费的开源文档数据库。它属于一个名为 NoSQL 的数据库系列，它与传统的基于表的 SQL 数据库（如 MySQL 和 PostgreSQL ）不同。

在 MongoDB 中，数据存储在灵活的、类 JSON 的文档中，其中字段可能因文档而异。它不需要预定义的模式，数据结构可以随时间更改。

在本教程中，我们将介绍从官方 MongoDB 存储库在 Ubuntu 18.04 计算机上安装和配置最新版 MongoDB Community Edition 的过程。

先决条件

在继续本教程之前，请确保以具有 sudo 权限的用户身份登录。

安装 MongoDB

在撰写本文时，最新版本的 MongoDB 是 4.0 版。

在继续安装过程之前，请转到 MongoDB 文档中的 Install on Ubuntu 部分，并检查是否有新版本可用。

以下步骤描述了如何在 Ubuntu 服务器上安装 MongoDB ：

  1. 启用 MongoDB 存储库

    首先，使用以下命令将 MongoDB GPG 密钥添加到系统：

    sudo apt-key adv --keyserver hkp://keyserver.ubuntu.com:80 --recv 9DA31620334BD75D9DCB49F368818C72E52529D4

    导入密钥后，添加 MongoDB 存储库：

    sudo add-apt-repository 'deb [arch=amd64] https://repo.mongodb.org/apt/ubuntu bionic/mongodb-org/4.0 multiverse'

    如果收到错误消息， add-apt-repository command not found 则说明您需要安装该 software-properties-common 软件包。

    要安装旧版本的 MongoDB ，只需替换 4.0 为您的想安装的版本即可。

  2. 安装 MongoDB

    为了能够从 MongoDB 存储库安装软件包，您需要首先更新软件包列表：

    sudo apt update

    现在已启用存储库并更新了包列表，您可以使用 apt 包管理器安装 mongodb-org 包：

    sudo apt install mongodb-org

    以下软件包将作为 mongodb-org 软件包的一部分安装在您的系统上：

    • mongodb-org-server- mongod 守护程序和相应的 init 脚本和配置。
    • mongodb-org-mongos- mongos 守护进程。
    • mongodb-org-shell - mongo shell 是 MongoDB 的交互式 JavaScript 接口。它用于通过命令行执行管理任务。
    • mongodb-org-tools - 包含几个 MongoDB 工具，用于导入和导出数据，统计信息以及其他实用程序。

  3. 启动 MongoDB

    安装完成后，通过键入以下内容启动 MongoDB 守护程序并启用随机自启动：

    sudo systemctl start mongod
sudo systemctl enable mongod
    1. 验证 MongoDB 安装

    要验证安装是否已成功完成，我们要使用 mongo 工具连接到 MongoDB 数据库服务器并打印连接状态：

    mongo --eval 'db.runCommand({ connectionStatus: 1 })'

    输出将如下所示：

    MongoDB shell version v4.0.2
connecting to: mongodb://127.0.0.1:27017
MongoDB server version: 4.0.2
{
  "authInfo" : {
    "authenticatedUsers" : [ ],
    "authenticatedUserRoles" : [ ]
  },
  "ok" : 1
}

    字段 ok 的值为 1 表示成功。

    配置 MongoDB

    MongoDB 使用 YAML 格式的配置文件 /etc/mongod.conf 。您可以通过编辑此文件来配置 MongoDB 实例。

    默认配置设置足以满足大多数用户的需求。但是，对于生产环境，建议把安全部分的注释取消并启用授权，如下所示：

    security:
authorization: enabled

选项 authorization 启用基于角色的访问控制(RBAC)，以控制用户对数据库资源和操作的访问。如果禁用此选项，则每个用户都可以访问所有数据库并执行任何操作。

在更改 MongoDB 配置文件后，使用以下命令重新启动 mongod 服务：

sudo systemctl restart mongod

要查找有关 MongoDB 4.0 中可用配置选项的更多信息，请访问配置文件选项文档页面。

创建管理 MongoDB 用户

如果启用了 MongoDB 身份验证，请创建一个管理 MongoDB 用户，该用户将用于访问和管理 MongoDB 实例。

首先访问 mongo shell ：

mongo

进入 MongoDB shell 后，键入以下命令以连接到 admin 数据库：

use admin
switched to db admin

发出以下命令以创建具有 userAdminAnyDatabase 角色的新用户 mongoAdmin：

db.createUser(
  {
    user: "mongoAdmin", 
    pwd: "changeMe", 
    roles: [ { role: "userAdminAnyDatabase", db: "admin" } ]
  }
)
Successfully added user: {
	"user" : "mongoAdmin",
	"roles" : [
		{
			"role" : "userAdminAnyDatabase",
			"db" : "admin"
		}
	]
}

您可以根据需要为管理 MongoDB 的用户命名。

退出 mongo shell ：

quit()

要测试更改，请使用先前创建的管理用户访问 mongo shell ：

mongo -u mongoAdmin -p --authenticationDatabase admin
use admin
switched to db admin

现在，打印用户：

show users
{
	"_id" : "admin.mongoAdmin",
	"user" : "mongoAdmin",
	"db" : "admin",
	"roles" : [
		{
			"role" : "userAdminAnyDatabase",
			"db" : "admin"
		}
	],
	"mechanisms" : [
		"SCRAM-SHA-1",
		"SCRAM-SHA-256"
	]
}

您也可以尝试访问没有任何参数的 mongo shell （只需键入 mongo ），看看是否可以使用与上面相同的命令列出用户。

结论

您已经学习了如何在 Ubuntu 18.04 服务器上安装和配置 MongoDB 4.0 。

有关此主题的更多信息，请参阅 The MongoDB 4.0 手册。
