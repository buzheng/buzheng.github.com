如何在 CentOS 7 上安装 Git

April 5, 2019

本教程将引导您在 CentOS 7 安装 Git 并完成基本的配置。

Git 是最受欢迎的版本控制系统，已被数十万个项目使用。 Git 允许您跟踪代码更改，恢复到以前的阶段，在多个分支上同时工作并与您的开发人员协作。 Git 最初是由 Linux 内核的创建者 Linus Torvalds 开发的。

在撰写本文时，默认的 CentOS 7 存储库中可用的 Git 版本 1.8.3 已经过时了。

安装最新版 Git (v2.18) 的最简单方法是使用 Wandisco 存储库中的 yum 软件包管理工具进行安装。

另一种选择是从源代码编译 Git ，这将允许您安装最新的 Git 版本并自定义构建选项，但您将无法通过 yum 包管理器维护 Git 安装。

先决条件

在继续本教程之前，请确保以具有 sudo 权限的用户身份登录。

在 CentOS 7 上安装 Git

按照以下步骤在 CentOS 7 系统上安装最新的 Git 版本：

  1. 启用 Wandisco GIT 存储库

    要启用 Wandisco GIT 存储库，请打开文本编辑器并在 /etc/yum.repos.d/ 目录中创建一个新的 YUM 存储库配置文件 wandisco-git.repo：

    文件: /etc/yum.repos.d/wandisco-git.repo

    [wandisco-git]
name=Wandisco GIT Repository
baseurl=http://opensource.wandisco.com/centos/7/git/$basearch/
enabled=1
gpgcheck=1
gpgkey=http://opensource.wandisco.com/RPM-GPG-KEY-WANdisco

    使用以下命令导入存储库 GPG 密钥：

    sudo rpm --import http://opensource.wandisco.com/RPM-GPG-KEY-WANdisco

  2. 在 CentOS 上安装 Git

    添加存储库后，要安装最新版本的 Git ，请运行以下命令：

    sudo yum install git
    1. 验证安装

    要验证安装，请输入以下命令打印 Git 版本的：

    git --version
    git version 2.18.0

    从上面的输出可以看出，我们在 CentOS 系统上成功安装了 Git 版本 2.18.0 。

    1. 配置 Git

    现在您已经安装了 Git ，最好设置在您提交代码更改时使用的个人信息：

    git config --global user.name "Your Name"
git config --global user.email "youremail@yourdomain.com"

    使用以下命令验证配置更改：

    git config --list
    user.name=Your Name
user.email=youremail@yourdomain.com

    配置存储在 ~/.gitconfig 文件中，以下显示文件的内容：

    [user]
    name = Your Name
    email = youremail@yourdomain.com

    要进一步更改 Git 配置，您可以使用 git config 命令或手动编辑 ~/.gitconfig 文件。

    结论

    我们已经向您展示了如何在 CentOS 7 上安装 Git 。您现在应该访问 Pro Git 书籍的在线版本，了解有关如何使用 Git 的更多信息。

centos git
