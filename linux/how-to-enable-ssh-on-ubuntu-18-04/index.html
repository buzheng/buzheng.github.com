如何在 Ubuntu 18.04 上启用 SSH

2019年5月29日 • 约 1400 字 , 预计阅读时间 5 分钟

Secure Shell(SSH) 是一种加密网络协议，用于客户端和服务器之间的安全连接。

在本教程中，我们将向您展示如何在 Ubuntu Desktop 计算机上启用 SSH 。启用 SSH 将允许您远程连接到 Ubuntu 计算机并安全地传输文件或执行管理任务。

先决条件

在继续本教程之前，请确保以具有 sudo 权限的用户身份登录。

在 Ubuntu 上启用 SSH

默认情况下，在 Ubuntu 桌面系统中不安装 SSH 服务器，但可以从主 Ubuntu 存储库轻松安装。

要在 Ubuntu 系统上安装和启用 SSH ，请完成以下步骤：

  1. 使用 Ctrl+Alt+T 键盘快捷键或单击终端图标打开终端，然后键入以下内容安装 openssh-server 软件包：

    sudo apt update
sudo apt install openssh-server

    出现提示时输入密码，然后输入 Y 以继续安装。

    1. 安装完成后， SSH 服务将自动启动。要验证安装是否成功并且 SSH 服务正在运行，请键入以下命令，该命令将打印 SSH 服务器状态：
    sudo systemctl status ssh

    你应该看到类似的东西 Active: active (running) ：

    按 q 返回命令行提示符。

现在 SSH 已在 Ubuntu 系统上安装并运行，您可以通过 SSH 从任何远程计算机连接到它。 Linux 和 macOS 系统默认安装了 SSH 客户端。如果要从 Windows 计算机进行连接，则可以使用 PuTTY 等 SSH 客户端。

通过 LAN 连接到 SSH

要通过 LAN 连接到 Ubuntu 计算机，只需输入以下命令：

ssh username@ip_address

username 使用实际用户名和 ip_address 安装 SSH 的 Ubuntu 计算机的 IP 地址进行更改。

如果您不知道您的 IP 地址，可以使用以下 ip a 命令轻松找到它：

ip a

从输出中可以看出，系统 IP 地址是 192.168.121.111 。

找到 IP 地址后，返回到您尝试登录的远程计算机并输入以下命令：

ssh buzheng@192.168.121.111

当您第一次通过 SSH 连接时，您将看到如下所示的消息：

The authenticity of host '192.168.121.111 (192.168.121.111)' can't be established.
ECDSA key fingerprint is SHA256:Vybt22mVXuNuB5unE++yowF7lgA/9/2bLSiO3qmYWBY.
Are you sure you want to continue connecting (yes/no)?

键入 yes 你会被提示输入您的密码。

Warning: Permanently added '192.168.121.111' (ECDSA) to the list of known hosts.
buzheng@192.168.121.111's password:

输入密码后，您将看到类似下面的消息。

Welcome to Ubuntu 18.04.1 LTS (GNU/Linux 4.15.0-33-generic x86_64)

 * Documentation:  https://help.ubuntu.com
 * Management:     https://landscape.canonical.com
 * Support:        https://ubuntu.com/advantage

...

您现在已登录到 Ubuntu 计算机。

通过 Internet 连接到 SSH

要通过 Internet 连接到您的 Ubuntu 计算机，您需要知道您的公共 IP 地址，并将路由器配置为接受端口 22 上的数据并将其发送到运行 SSH 的 Ubuntu 计算机。

要确定您尝试 SSH 到的计算机的公共 IP 地址，只需访问以下 URL : https://ifconfig.co/ip.

在设置端口转发时，每个路由器都有不同的设置端口转发的方法。您应该查阅有关如何设置端口转发的路由器文档。简而言之，您需要输入要求进行请求的端口号（默认 SSH 端口为 22 ）以及您之前（使用 ip a 命令）找到运行 SSH 的计算机的专用 IP 地址。

找到 IP 地址并配置路由器后，您可以输入以下命令登录：

ssh username@public_ip_address

如果您将机器暴露在 Internet 上，最好实施一些安全措施。最基本的一种是将路由器配置为接受非标准端口上的 SSH 流量，并将其转发到运行 SSH 服务的计算机上的端口 22 。

您还可以设置基于 SSH 密钥的身份验证，并在不输入密码的情况下连接到 Ubuntu 计算机。

在 Ubuntu 上禁用 SSH

如果由于某种原因你想在你的 Ubuntu 机器上禁用 SSH ，你可以通过运行以下命令来停止 SSH 服务：

sudo systemctl stop ssh

要再次启动它：

sudo systemctl start ssh

要在系统引导运行期间禁用 SSH 服务，请执行以下操作：

sudo systemctl disable ssh

要再次启用它，请键入：

sudo systemctl enable ssh

结论

您已经学习了如何在 Ubuntu 18.04 上安装和启用 SSH 。您现在可以登录到您的计算机添加执行常见的 sysadmin 任务以及命令提示符。您可能还需要设置基于 SSH 密钥的身份验证并连接到 Linux 服务器而无需输入密码。

默认情况下， SSH 侦听端口是 22， 更改默认 SSH 端口可以降低自动攻击的风险，从而为服务器增加额外的安全层。

如果您要管理多个系统，则可以通过在 SSH 配置文件中定义所有连接来简化工作流程。

有关如何配置 SSH 服务器的更多信息，请阅读 Ubuntu 的 SSH/OpenSSH/Configuring 指南和官方 SSH 手册页。
