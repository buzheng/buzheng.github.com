如何在CentOS 7上安装和使用Docker Compose

2019年5月9日 • 约 2100 字 , 预计阅读时间 7 分钟

Docker Compose是一个允许您定义和运行多容器Docker应用程序的工具。

使用Compose，您可以在单个YAML文件中定义应用程序的服务，网络和卷，然后使用单个命令旋转应用程序。

Compose可用于不同目的，例如单主机应用程序部署，自动化测试和本地开发。

本教程将指导您在CentOS 7上安装最新版本的Docker Compose。我们还将介绍基本的Docker Compose概念和命令。

先决条件

在继续本教程之前，请确保满足以下先决条件：

  • 以具有sudo权限的用户身份登录。
  • 已安装你的CentOS 7泊坞窗系统。

在CentOS上安装Docker Compose

在CentOS 7上安装Docker Compose的推荐方法是从Docker的GitHub存储库下载Compose二进制文件。

在撰写本文时，Docker Compose的最新稳定版本是版本1.23.1。在下载Compose二进制文件之前，请访问GitHub上的Compose存储库版本页面，检查是否有可供下载的新版本。

完成以下步骤在CentOS 7上安装Docker Compose：

  1. 首先/usr/local/bin使用以下curl命令将Docker Compose二进制文件下载到目录中：

    sudo curl -L "https://github.com/docker/compose/releases/download/1.23.1/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

  2. 下载完成后，键入以下命令使二进制文件可执行：

    sudo chmod +x /usr/local/bin/docker-compose

  3. 要验证安装类型，请使用以下命令打印Compose版本：

    docker-compose --version

    输出看起来像这样：

    docker-compose version 1.23.1, build b02f1306

Docker Compose入门

在本节中，我们将展示如何使用Docker Compose在CentOS 7计算机上运行WordPress堆栈。

首先为项目创建一个新目录并将其更改为：

mkdir my_app && cd my_app

接下来，docker-compose.yml在项目目录中创建一个名为的文件：

nano docker-compose.yml

粘贴以下内容：

泊坞窗，compose.yml

version: '3.3'

services:
  db:
    image: mysql:5.7
    restart: always
    volumes:
      - db_data:/var/lib/mysql
    environment:
      MYSQL_ROOT_PASSWORD: password
      MYSQL_DATABASE: wordpress

  wordpress:
    image: wordpress
    restart: always
    volumes:
      - ./wp_data:/var/www/html
    ports:
      - "8080:80"
    environment:
      WORDPRESS_DB_HOST: db:3306
      WORDPRESS_DB_NAME: wordpress
      WORDPRESS_DB_USER: root
      WORDPRESS_DB_PASSWORD: password
    depends_on:
       - db

volumes:
    db_data:
    wp_data:

让我们逐行分析代码。

在第一行中，我们指定了Compose文件版本。有几种不同版本的Compose文件格式，支持特定的Docker版本。

接下来我们定义两个服务，db和wordpress。每个服务都运行一个映像，并在运行docker-compose时创建一个单独的容器。

该db服务：

  • 使用mysql:5.7图像。在您的系统上没有图像Compose将从Docker Hub公共存储库中提取它。
  • 使用重启always策略，该策略将指示容器始终重新启动。
  • 创建命名卷db_data以保留数据库。
  • 定义mysql:5.7图像的环境变量。

该wordpress服务：

  • 使用wordpress图像。在您的系统上没有图像Compose将从Docker Hub公共存储库中提取它。
  • 使用重启always策略，该策略将指示容器始终重新启动。
  • wp_data将主机上的目录安装到/var/lib/mysql容器内。
  • 将容器上的公开端口80转发到主机上的端口8080。
  • 定义wordpress图像的环境变量。
  • 该depends_on指令定义了两个服务之间的依赖关系。在这个例子中，db将在之前启动wordpress。

从项目目录中，使用以下命令启动WordPress应用程序：

docker-compose up

输出应该如下所示：

...
wordpress_1  | [Sat Oct 13 21:30:48.286382 2018] [mpm_prefork:notice] [pid 1] AH00163: Apache/2.4.25 (Debian) PHP/7.2.10 configured -- resuming normal operations
wordpress_1  | [Sat Oct 13 21:30:48.286425 2018] [core:notice] [pid 1] AH00094: Command line: 'apache2 -D FOREGROUND'

Compose将拉出两个图像，启动两个容器并wp_data在项目目录中创建目录。

打开浏览器，输入http://0.0.0.0:8080/，您将看到Wordpress安装屏幕。

此时，Wordpress应用程序已启动并运行，您可以开始使用主题或插件。

要停止撰写新闻CTRL+C。

如果要以分离模式启动Compose，请使用-d标志：

docker-compose up -d

要检查正在运行的服务，请使用以下ps选项

docker-compose ps

       Name                     Command               State          Ports        
----------------------------------------------------------------------------------
my_app_db_1          docker-entrypoint.sh mysqld      Up      3306/tcp, 33060/tcp 
my_app_wordpress_1   docker-entrypoint.sh apach ...   Up      0.0.0.0:8080->80/tcp

当Compose以分离模式运行以停止服务使用时：

docker-compose stop

要完全删除容器，请使用以下down选项：

docker-compose down

通过--volumes交换机也将删除数据卷：

docker-compose down --volumes

卸载Docker Compose

如果要卸载Docker Compose，只需使用以下命令删除二进制文件：

sudo rm /usr/local/bin/docker-compose

结论

您已经学习了如何在CentOS 7上安装和使用Docker Compose。
