如何在 Ubuntu 18.04 上启用和设置无人值守自动安全更新

2019年3月10日 • 约 1100 字 , 预计阅读时间 5 分钟

定期更新 Ubuntu 系统是保证系统安全性最重要的方法之一。如果不使用最新的安全修补程序更新操作系统的软件包，则计算机很容易受到攻击。

如果您手动管理多个 Ubuntu 机器，更新系统软件包可能非常耗时。即使您管理单个 Ubuntu 安装，有时您可能会忽略重要更新。这是无人值守自动更新的便利之处。

在本教程中，我们将介绍如何在 Ubuntu 18.04 上安装和配置无人值守自动更新。相同的步骤适用于 Ubuntu 16.04 和任何基于 Ubuntu 的发行版，包括 Kubuntu ， Linux Mint 和 Elementary OS 。

先决条件

在继续本教程之前，请确保以具有 sudo 权限的用户身份登录。

安装 unattended-upgrades 升级包

这个包 unattended-upgrades 用于自动安装更新的软件包。有可能这个软件包已经安装在您的 Ubuntu 系统上，如果没有，您可以通过在终端中输入以下命令来安装软件包：

sudo apt install unattended-upgrades

安装完成后，该服务并自动启动并开启了随机自启动。您可以输入以下命令进行验证：

systemctl status unattended-upgrades

● unattended-upgrades.service - Unattended Upgrades Shutdown
   Loaded: loaded (/lib/systemd/system/unattended-upgrades.service; enab
   Active: active (running) since Sun 2019-03-10 07:52:08 UTC; 2min 35s 
     Docs: man:unattended-upgrade(8)
   CGroup: /system.slice/unattended-upgrades.serviceq

配置无人值守自动更新

我们可以通过编辑 /etc/apt/apt.conf.d/50unattended-upgrades 文件来配置 unattended-upgrades 。默认配置一般适用于大多数用户，但您可以打开文件并根据需要进行更改。

无人值守升级包，可以配置为更新所有包或仅更新安全更新。第一部分定义了哪些类型的包将自动更新。默认情况下，它将仅安装安全更新，如果要从其他存储库启用更新，则可以通过从行开头删除双斜杠 // 来取消注释相应的存储库。 配置文件中每行 // 之后的任何内容都是注释，不会被读取。

文件 /etc/apt/apt.conf.d/50unattended-upgrades

Unattended-Upgrade::Allowed-Origins {
  "${distro_id}:${distro_codename}";
	"${distro_id}:${distro_codename}-security";
	// Extended Security Maintenance; doesn't necessarily exist for
	// every release and this system may not have it installed, but if
	// available, the policy for updates is such that unattended-upgrades
	// should also install from here by default.
	"${distro_id}ESM:${distro_codename}";
//	"${distro_id}:${distro_codename}-updates";
//	"${distro_id}:${distro_codename}-proposed";
//	"${distro_id}:${distro_codename}-backports";
};

如果由于任何原因您想要禁用某些软件包自动更新，只需添加到软件包黑名单中：

Unattended-Upgrade::Package-Blacklist {
//      "vim";
//      "libc6";
//      "libc6-dev";
//      "libc6-i686";
};

如果由于某种原因自动更新出现问题，您可能还希望收到电子邮件通知。为此，请取消以下两行注释并输入您的电子邮件地址。确保系统上有一个可以发送电子邮件的工具，例如 mailx 或 postfix 。

Unattended-Upgrade::Mail "your@email.com";
Unattended-Upgrade::MailOnlyOnError "true";

启用无人值守自动更新

要启用自动更新，您需要确保 apt 配置文件 /etc/apt/apt.conf.d/20auto-upgrades 中至少包含以下两行，默认情况下应包含这两行：

APT::Periodic::Update-Package-Lists "1";
APT::Periodic::Unattended-Upgrade "1";

以上配置会每天更新包列表，并安装可用更新。

您还可以在文件 /etc/apt/apt.conf.d/20auto-upgrades 添加以下行，该行每 7 天清理一次本地下载的档案。

APT::Periodic::AutocleanInterval "7";

启用/禁用自动更新的另一种方法是运行以下命令，该命令将修改（如果不存在或创建）文件 /etc/apt/apt.conf.d/20auto-upgrades 。

sudo dpkg-reconfigure -plow unattended-upgrades

测试

要测试自动升级是否工作请直接测试运行，如下面命令所示：

sudo unattended-upgrades --dry-run --debug

输出应该如下所示：

...
pkgs that look like they should be upgraded: 
Fetched 0 B in 0s (0 B/s)                                                                                             
fetch.run() result: 0
blacklist: []
whitelist: []
No packages found that can be upgraded unattended and no pending auto-removals

无人值守自动升级的历史记录记录在 /var/log/unattended-upgrades/unattended-upgrades.log 文件中。

结论

在本教程中，您学习了如何自动安装更新的软件包，并保持系统启用最新的安全更新。
