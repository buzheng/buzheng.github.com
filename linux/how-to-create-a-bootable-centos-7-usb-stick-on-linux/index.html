如何在 Linux 上创建可启动的 CentOS 7 U 盘

2019年6月12日 • 约 800 字 , 预计阅读时间 4 分钟

在本教程中，我们将解释如何从 Linux 终端创建可引导的 CentOS U 盘。您可以使用此 U 盘在任何支持从 USB 启动的计算机上启动并测试或安装 CentOS 。

先决条件

  • 一个 8GB 或更大的 U 盘
  • 计算机运行任何 Linux 发行版

下载 CentOS ISO 文件

从 CentOS 下载页面下载 CentOS ISO 文件，您可以在其中选择“ DVD ISO ”和“最小 ISO ”。

最小安装映像仅包含具有功能 CentOS 系统所需的软件包。 DVD 映像包含可以使用安装程序安装的所有软件包。

您最好下载 “DVD ISO” 版本。

在 Linux 上创建可启动的 CentOS 7 U 盘

虽然有许多不同的 GUI 工具允许您将 ISO 映像闪存到 USB 驱动器，但在本教程中，我们将使用该 dd 命令创建可启动的 CentOS 7 USB 闪存盘。

在 Linux 上创建可启动的 CentOS 7 USB Stick 是一个快速简便的过程，只需按照下面的详细步骤操作即可。

1、 将 USB 闪存盘插入 USB 端口。

2、 使用以下 lsblk 命令找出 USB 驱动器的名称：

lsblk

输出将如下所示：

NAME        MAJ:MIN RM   SIZE RO TYPE MOUNTPOINT 
sda           8:0    0 465.8G  0 disk  
 └─sda1       8:1    0 465.8G  0 part /data 
sdx           8:16   1   7.5G  0 disk  
 └─sdx1       8:17   1   7.5G  0 part /run/media/linuxize/Kingston 
nvme0n1     259:0    0 232.9G  0 disk  
 ├─nvme0n1p1 259:1   0   512M  0 part /boot 
 ├─nvme0n1p2 259:2   0    16G  0 part [SWAP] 
 └─nvme0n1p3 259:3   0 216.4G  0 part /

在我们的例子中， USB 设备的名称是 /dev/sdx， 但这可能会跟您的系统上显示的不同。

3、 在大多数 Linux 发行版中， USB 闪存驱动器将在插入时自动安装。在刷入镜像之前，请确保未安装 USB 设备。要执行此操作，请使用 umount 命令后跟安装它的目录(安装点)或设备名称：

sudo umount /dev/sdx1

4、 最后一步是将 CentOS ISO 映像刷新到 USB 驱动器。确保使用您自己的驱动器进行更换 /dev/sdx ，并且不要附加分区号。另外，替换 /path/to/CentOS-7-x86_64-DVD-1810.iso 为 ISO 文件的路径。如果您使用 Web 浏览器下载了该文件，则应将其存储 Downloads 在您的用户帐户中的文件夹中。

sudo dd bs=4M if=/path/to/CentOS-7-x86_64-DVD-1810.iso of=/dev/sdx status=progress oflag=sync

该命令将在刷入镜像时显示进度条。

此过程可能需要几分钟，具体取决于 ISO 文件的大小和 U 盘的写入速度。完成后，您将看到如下内容：

1094+0 records in 
1094+0 records out 
4588568576 bytes (4.6 GB) copied, 30.523 s, 150 MB/s

就是这样，此时你的 U 盘上有一个可启动的 CentOS 。

结论

在本文中，您已经学习了如何从 Linux 命令行创建可引导的 CentOS U 盘。
