如何在Debian 9上安装和使用Composer

2019年5月9日 • 约 2100 字 , 预计阅读时间 7 分钟

Composer是PHP的依赖管理器（类似于Node.js的npm或Python的pip）。Composer将引入您项目所依赖的所有必需PHP包并为您管理它们。

本教程提供了在Debian 9系统上安装Composer所需的步骤。我们还将介绍如何使用Composer创建和管理PHP项目。

先决条件

在继续学习本教程之前，请确保以具有sudo权限的用户身份登录，并且在Debian 9系统上安装了PHP。

安装Composer

按照以下步骤在Debian系统上安装Composer：

  1. 首先更新软件包索引并使用以下命令安装必要的软件包：

    sudo apt updatesudo apt install php-cli php-zip wget unzip
    1. 安装依赖项后，使用php cli toll下载Composer安装脚本：
    php -r "copy('https://getcomposer.org/installer', 'composer-setup.php');"

    上面的命令将下载composer-setup.php当前工作目录中的文件。

  2. 我们将通过将脚本SHA-384哈希与Composer Public Keys / Signatures页面上找到的最新安装程序哈希进行比较来验证脚本数据的完整性。

    运行以下wget命令从Composer的Github页面下载最新Composer安装程序的预期签名，并将其存储在名为的变量中HASH：

    HASH="$(wget -q -O - https://composer.github.io/installer.sig)"

    要验证安装脚本是否已损坏，请将以下代码复制并粘贴到控制台中：

    php -r "if (hash_file('SHA384', 'composer-setup.php') === '$HASH') { echo 'Installer verified'; } else { echo 'Installer corrupt'; unlink('composer-setup.php'); } echo PHP_EOL;"

    如果哈希匹配，您将看到以下输出：

    Installer verified

    如果哈希值不匹配，你会看到Installer corrupt。在这种情况下，您需要重新下载Composer安装脚本并仔细检查$HASH变量的值echo $HASH。验证安装程序后，您可以继续执行下一步。

    1. 以下命令将在/usr/local/bin目录中安装Composer ：
    sudo php composer-setup.php --install-dir=/usr/local/bin --filename=composer
    All settings correct for using Composer
Downloading...

Composer (version 1.7.2) successfully installed to: /usr/local/bin/composer
Use it: php /usr/local/bin/composer

    此时，您已在Debian系统上安装了Composer。它作为系统范围的命令安装，可供所有用户使用。

    1. 要验证安装，只需键入：
    composer

    上面的命令将打印Composer的版本，命令和参数。

    Composer入门

    现在您已经在Debian系统上安装了Composer，我们将向您展示如何使用Composer创建PHP项目。

    首先创建一个目录，该目录将是项目根目录并保存composer.json文件。此文件描述了您的PHP项目，包括PHP依赖项和其他元数据。

    创建项目目录并切换到它：

    mkdir ~/my-first-composer-projectcd ~/my-first-composer-project

接下来，我们将composer.json使用该composer require <package name>命令初始化一个新文件，并指定我们要下载的包。在此示例中，我们将创建一个示例应用程序，该应用程序将使用名为carbon的包打印当前时间。

运行以下命令以初始化新composer.json文件并安装碳包：

composer require nesbot/carbon

Composer将创建一个composer.json文件并下载并安装carbon及其所有依赖项。

如果你与你的列出项目的目录ls命令，你会看到它包含两个文件composer.json和composer.lock，和vendor目录。

ls -l
-rw-r--r-- 1 linuxize users   60 Oct 16 21:02 composer.json
-rw-r--r-- 1 linuxize users 6851 Oct 16 21:02 composer.lock
drwxr-xr-x 5 linuxize users 4096 Oct 16 21:02 vendor

  • vendor - 存储项目依赖项的目录。

  • composer.lock - 包含所有已安装软件包列表的文件，包括确切的软件包版本。

  • composer.json - 描述PHP项目和所有PHP依赖项的文件。

您可以在Composer存储库Packagist中搜索PHP包。

Composer提供自动加载功能，允许我们在不需要require或include文件的情况下使用PHP类。

创建一个名为的文件testing.php并粘贴以下代码：

<?php

require __DIR__ . '/vendor/autoload.php';

use Carbon\Carbon;

printf("Now: %s", Carbon::now());

让我们逐行分析代码。

在打开php标记之后的第一行中，我们包含了vendor/autoload.phpComposer自动生成的文件。此文件将自动加载所有必需的库。

接下来我们是别名Carbon\Carbon，Carbon最后一行是使用Carbon now方法打印当前时间。

键入以下命令运行脚本：

php testing.php

输出应如下所示：

Now: 2018-10-16 21:08:45

3.使用Composer更新包

要更新PHP包，只需运行：

composer update

上面的命令将检查已安装软件包的较新版本，如果找到较新版本且版本约束与指定的版本约束匹配，则composer.jsonComposer将更新软件包。
