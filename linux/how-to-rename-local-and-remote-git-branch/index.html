如何重命名本地和远程Git分支

May 9, 2019

您正在与一组人合作开发项目，并且已经为git分支定义了命名约定。您创建了一个新分支，将更改推送到远程存储库并意识到您的分支名称不正确。

幸运的是，Git允许您使用该-m选项非常轻松地重命名分支。

本指南将解释如何重命名本地和远程Git分支。

重命名Git Branch

按照以下步骤重命名本地和远程Git分支：

  1. 首先切换到要重命名的本地分支：

    git checkout <old_name>
    1. 键入以下命令重命名本地分支：
    git branch -m <new_name>

  2. 如果您已将<old_name>分支推送到远程存储库，请删除<old_name>远程分支：

    git push origin --delete <old_name>
    1. 最后推送<new_name>本地分支并重置上游分支
    git push origin -u <new_name>

而已。此时，您已成功重命名本地和远程Git分支。

结论

重命名本地Git分支只是一个命令的问题。但是，您无法直接重命名远程分支，需要将其删除，然后重新推送重命名的本地分支。
