如何在 Ubuntu 18.04 上安装和配置 Gogs

April 26, 2019

Gogs 是一个用 Go 编写的自托管开源 git 服务器。它包括存储库文件编辑器，项目问题跟踪，并且内置维基。

Gogs 是一个轻量级应用程序，内存占用空间更小，可以安装在低功耗系统上。如果您正在寻找 Gitlab 的替代品，而且您不需要 Gitlab 提供的所有功能，那么您一定要尝试 Gogs 。

本教程介绍了在 Ubuntu 18.04 上安装和配置 Gogs 的步骤。相同的说明适用于 Ubuntu 16.04 和任何其他基于 Ubuntu 的发行版。

在你开始之前

Gogs 可以使用 SQLite ， PostgreSQL 或 MySQL / MariaDB 数据库来存储其所有数据。

在本教程中，我们将使用 SQLite 作为首选数据库。如果系统上未安装 SQLite ，则可以通过键入以下命令来安装它：

sudo apt install sqlite3

为了增加安全层，建议设置基本防火墙。您可以按照指南(在Ubuntu 18.04 如何使用 UFW 设置防火墙中)中的步骤进行操作。

安装 Gogs

我们将从二进制文件安装 Gogs 。安装是一个非常简单的过程。

安装 Git

第一步是在服务器上安装 Git 。为此，请刷新本地程序包索引并通过以 sudo 用户身份运行以下命令来安装 git 程序包：

sudo apt update
sudo apt install git

通过显示 Git 版本来验证安装：

git --version
git version 2.17.1

创建一个 Git 用户

通过键入以下内容创建一个新的系统用户来运行 Gogs 服务：

sudo adduser --system --group --disabled-password --shell /bin/bash --home /home/git --gecos 'Git Version Control' git

该命令将创建用户并将主目录设置为 /home/git 。输出结果如下所示：

Adding system user `git' (UID 111) ...
Adding new group `git' (GID 116) ...
Adding new user `git' (UID 111) with group `git' ...
Creating home directory `/home/git' ...

下载 Gogs 二进制文件

访问 Gogs 下载页面并下载适用于您的体系结构的最新二进制文件。在撰写本文时，最新版本为 0.11.86 ，如果有新版本可用，请在下面的命令中更改变量 VERSION。

使用以下 wget 命令下载 Gogs 存档到 /tmp 目录中：

VERSION=0.11.86wget https://dl.gogs.io/${VERSION}/gogs_${VERSION}_linux_amd64.tar.gz -P /tmp

下载完成后，解压缩 Gogs 的 tar.gz 文件并将其移动到 /home/git 目录：

sudo tar xf /tmp/gogs_*_linux_amd64.tar.gz -C /home/git

运行以下命令更改 Gogs 安装目录的所有权为 git 用户和组：

sudo chown -R git: /home/git/gogs

创建一个 systemd 单元文件

Gogs 附带一个 Systemd 单元文件，该文件已配置为与我们的设置相匹配。

键入以下命令将文件复制到目录 /etc/systemd/system/ ：

sudo cp /home/git/gogs/scripts/systemd/gogs.service /etc/systemd/system/

完成后，启动并启用 Gogs 服务：

sudo systemctl start gogs
sudo systemctl enable gogs

验证服务是否已成功启动：

* gogs.service - Gogs
   Loaded: loaded (/etc/systemd/system/gogs.service; enabled; vendor preset: enabled)
   Active: active (running) since Thu 2019-04-25 04:13:44 PDT; 9s ago
 Main PID: 14376 (gogs)
    Tasks: 8 (limit: 2319)
   CGroup: /system.slice/gogs.service
           `-14376 /home/git/gogs/gogs web

使用 Web 安装程序安装 Gogs

现在已经下载并运行 Gogs ，现在是时候通过 Web 界面完成安装了。

打开浏览器，键入 http://YOUR_DOMAIN_IR_IP:3000 并出现类似如下的屏幕：

数据库设置：

  • 数据库类型： SQLite3
  • 路径：使用绝对路径，  /home/git/gogs/gogs.db

应用程序常规设置

  • 应用程序名称：输入您的组织名称
  • 存储库根路径：保留默认值  /home/git/gogs-repositories
  • 运行用户： git
  • 域：输入您的域或服务器 IP 地址。
  • SSH 端口： 22 ，如果 SSH 正在侦听其他端口，请更改它
  • HTTP 端口： 3000
  • 应用程序 URL ：使用 http 和您的域或服务器 IP 地址。
  • 日志路径：保留默认值  /home/git/gogs/log

稍后您可以通过编辑 Gogs 配置文件来更改设置。

完成后点击“安装 Gogs ”按钮。安装是即时的，完成后您将被重定向到登录页面。

点击“立即注册”链接。

第一个注册用户将自动添加到 Admin 组。

现在， Gogs 已安装在您的 Ubuntu 机器上。

将 Nginx 配置为 SSL 终结代理

此步骤是可选的，但强烈建议您这样做。要将 Nginx 用作反向代理，，您需要具有指向服务器公共 IP 的域或子域。在本教程中，我们将使用 gogs.example.com 。

首先，按照以下指南安装 Nginx 并生成免费的 Let’s Encrypt SSL 证书：

  • 如何在 Ubuntu 18.04 上安装 Nginx
  • 在 Ubuntu 18.04 上使用 Let’s Encrypt 来保护 Nginx

完成后，打开文本编辑器并编辑域服务器块文件 /etc/nginx/sites-enabled/gogs.example.com：

sudo nano /etc/nginx/sites-enabled/gogs.example.com
server {
    listen 80;
    server_name gogs.example.com;

    include snippets/letsencrypt.conf;
    return 301 https://gogs.example.com$request_uri;
}

server {
    listen 443 ssl http2;
    server_name gogs.example.com;

    proxy_read_timeout 720s;
    proxy_connect_timeout 720s;
    proxy_send_timeout 720s;

    client_max_body_size 50m;

    # Proxy headers
    proxy_set_header X-Forwarded-Host $host;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_set_header X-Real-IP $remote_addr;

    # SSL parameters
    ssl_certificate /etc/letsencrypt/live/gogs.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/gogs.example.com/privkey.pem;
    ssl_trusted_certificate /etc/letsencrypt/live/gogs.example.com/chain.pem;
    include snippets/letsencrypt.conf;
    include snippets/ssl.conf;

    # log files
    access_log /var/log/nginx/gogs.example.com.access.log;
    error_log /var/log/nginx/gogs.example.com.error.log;

    # Handle / requests
    location / {
       proxy_redirect off;
       proxy_pass http://127.0.0.1:3000;
    }
}

不要忘记将 gogs.example.com 替换为 Gogs 真正的域名，并设置 SSL 证书文件的正确路径。

重新启动 Nginx 服务以使更改生效：

sudo systemctl restart nginx

接下来，我们需要更改 Gogs 域和根 URL 。为此，请打开配置文件 /home/git/gogs/custom/conf/app.ini 并编辑以下行：

sudo nano /home/git/gogs/custom/conf/app.ini
[server]
DOMAIN           = gogs.example.com
ROOT_URL         = https://gogs.example.com/

输入以下命令重新启动 Gogs 服务：

sudo systemctl restart gogs

此时，配置了 Gogs ，您可以在以下位置访问它：  https://gogs.example.com

配置电子邮件通知

为了让 Gog 能够发送通知电子邮件，您可以安装 Postfix 或使用一些邮件服务，如 SendGrid ， MailChimp ， MailGun 或 SES 。

要启用电子邮件通知，请打开配置文件 /home/git/gogs/custom/conf/app.ini 并编辑以下行：

sudo nano /home/git/gogs/custom/conf/app.ini
[mailer]
ENABLED = true
HOST    = SMTP_SERVER:SMTP_PORT
FROM    = SENDER_EMAIL
USER    = SMTP_USER
PASSWD  = YOUR_SMTP_PASSWORD

确保您输入正确的 SMTP 服务器信息。

重新启动 Gogs 服务以使更改生效：

sudo systemctl restart gogs

Gogs 还允许您通过创建 Web webhook 连接到 Slack  并向 Slack 通道发送通知。

升级 Gogs

要升级 Gogs ，需要执行几个手动步骤。

  1. 首先停止 Gogs 服务：

    sudo systemctl stop gogs
    1. 重命名 Gogs 安装目录。
    sudo mv /home/git/gogs{,_old}

  2. 下载最新的 Gogs 版本并将其移至 /home/git 目录：

    VERSION=<THE_LATEST_GOGS_VERSION>wget https://dl.gogs.io/${VERSION}/gogs_${VERSION}_linux_amd64.tar.gz -P /tmpsudo tar xf /tmp/gogs_*_linux_amd64.tar.gz -C /home/git

    确保您使用实际的 Gogs 版本来更改 VERSION 。

    1. 使用下面 rsync 命令 复制 custom ， data ， log 目录：
    sudo rsync -a /home/git/gogs_old/{custom,data,log} /home/git/gogs/

  3. 最后，启动 Gogs 服务：

    sudo systemctl restart gogs

    完成。

    结论

    本教程向您介绍了 Ubuntu 18.04 上 Gogs 的安装和配置。您现在可以创建第一个项目并开始使用新的 Gogs 服务器。
