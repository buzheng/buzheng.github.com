如何在 Ubuntu 18.04 上安装 Tomcat 9

May 13, 2019

Apache Tomcat 是 Java Servlet ， JavaServer Pages ， Java Expression Language 和 Java WebSocket 技术的开源实现。它是当今世界上应用最广泛的应用程序和 Web 服务器之一。 Tomcat 易于使用，并且具有强大的附加组件生态系统。

本教程将向您展示如何在 Ubuntu 18.04 上安装和配置 Tomcat 9 。相同的说明适用于 Ubuntu 16.04 和任何基于 Ubuntu 的发行版，包括 Linux Mint 和 Elementary OS 。

先决条件

为了能够在 Ubuntu 系统上安装软件包，您必须以具有 sudo 权限的用户身份登录。

第 1 步：安装 OpenJDK

Tomcat 需要安装 Java 。我们将安装 OpenJDK ，它是 Ubuntu 18.04 中的默认 Java 开发和运行时。

Java 的安装非常简单。首先更新包索引：

sudo apt update

运行以下命令安装 OpenJDK 包：

sudo apt install default-jdk

如果要使用 Oracle Java 而不是 OpenJDK ，请查看说明。

第 2 步：创建 Tomcat 用户

出于安全考虑，不应在 root 用户下运行 Tomcat 。我们将创建一个新的系统用户和组来运行 Tomcat 服务，用户的主目录是 /opt/tomcat ：

sudo useradd -r -m -U -d /opt/tomcat -s /bin/false tomcat

第 3 步：安装 Tomcat

我们从 Tomcat 9 下载页面下载最新的 Tomcat 9 二进制版本。

在撰写本文时，最新版本是 9.0.14 。在继续下一步之前，您应该检查下载页面以获取新版本。如果有新版本，则复制新的文件的链接到以下的命令中。

首先使用以下 wget 命令在目录中 /tmp 下载 Tomcat 存档：

wget http://www-eu.apache.org/dist/tomcat/tomcat-9/v9.0.14/bin/apache-tomcat-9.0.14.tar.gz -P /tmp

下载完成后，解压缩 Tomcat 存档并将其移动到 /opt/tomcat 目录：

sudo tar xf /tmp/apache-tomcat-9*.tar.gz -C /opt/tomcat

为了更好地控制 Tomcat 版本和更新，我们将创建一个 latest 指向 Tomcat 安装目录的符号链接：

sudo ln -s /opt/tomcat/apache-tomcat-9.0.14 /opt/tomcat/latest

如果以后要升级 Tomcat 安装，只需解压缩较新版本并更改符号链接以指向最新版本。

正如我们在上一节中提到的， Tomcat 将在 tomcat 用户下运行。该用户需要访问 tomcat 安装目录。

以下命令将更改目录所有权为用户和组tomcat：

sudo chown -RH tomcat: /opt/tomcat/latest

bin 目录中的脚本必须具有可执行标志：

sudo sh -c 'chmod +x /opt/tomcat/latest/bin/*.sh'

第 4 步：创建 systemd 单元文件

要将 Tomcat 作为服务运行，我们将创建一个新的单元文件。打开文本编辑器，在 /etc/systemd/system/ 目录中创建一个文件 tomcat.service ：

sudo nano /etc/systemd/system/tomcat.service

粘贴以下配置到文件 /etc/systemd/system/tomcat.service 中：

[Unit]
Description=Tomcat 9 servlet container
After=network.target

[Service]
Type=forking

User=tomcat
Group=tomcat

Environment="JAVA_HOME=/usr/lib/jvm/default-java"
Environment="JAVA_OPTS=-Djava.security.egd=file:///dev/urandom -Djava.awt.headless=true"

Environment="CATALINA_BASE=/opt/tomcat/latest"
Environment="CATALINA_HOME=/opt/tomcat/latest"
Environment="CATALINA_PID=/opt/tomcat/latest/temp/tomcat.pid"
Environment="CATALINA_OPTS=-Xms512M -Xmx1024M -server -XX:+UseParallelGC"

ExecStart=/opt/tomcat/latest/bin/startup.sh
ExecStop=/opt/tomcat/latest/bin/shutdown.sh

[Install]
WantedBy=multi-user.target

根据实际 Java 安装路径修改 JAVA_HOME 的值。

保存并关闭文件并通知 systemd 我们创建了一个新的单元文件：

sudo systemctl daemon-reload

通过执行以下命令启动 Tomcat 服务：

sudo systemctl start tomcat

使用以下命令检查服务状态：

sudo systemctl status tomcat
* tomcat.service - Tomcat 9 servlet container
   Loaded: loaded (/etc/systemd/system/tomcat.service; disabled; vendor preset: enabled)
   Active: active (running) since Wed 2018-09-05 15:45:28 PDT; 20s ago
  Process: 1582 ExecStart=/opt/tomcat/latest/bin/startup.sh (code=exited, status=0/SUCCESS)
 Main PID: 1604 (java)
    Tasks: 47 (limit: 2319)
   CGroup: /system.slice/tomcat.service

如果没有错误，则启用 Tomcat 服务在引导时自动启动：

sudo systemctl enable tomcat

第 5 步：调整防火墙

如果您的服务器受防火墙保护，并且您希望从本地网络外部访问 Tomcat 接口，则需要打开端口 8080。

要允许端口 8080 上的流量，请键入以下命令：

sudo ufw allow 8080/tcp

在生产环境中运行 Tomcat 应用程序时，很可能您将拥有负载平衡器或反向代理，最佳做法是仅将端口 8080 的访问权限限制到内部网络。

第 6 步：配置 Tomcat Web 管理界面

现在已经安装并运行Tomcat，下一步是创建一个可以访问Web管理界面的用户。

Tomcat 用户和角色在 tomcat-users.xml 文件中定义。此文件是一个模板，其中包含描述如何配置创建用户或角色的注释和示例。

sudo nano /opt/tomcat/latest/conf/tomcat-users.xml

要添加能够访问 Tomcat Web 界面 (manager-gui和admin-gui) 的新用户，我们需要在 tomcat-users.xml 文件中定义用户，如下所示。请务必将用户名和密码更改为更安全的内容：

编辑文件: /opt/tomcat/latest/conf/tomcat-users.xml

<tomcat-users>
<!--
    Comments
-->
   <role rolename="admin-gui"/>
   <role rolename="manager-gui"/>
   <user username="admin" password="admin_password" roles="admin-gui,manager-gui"/>
</tomcat-users>

默认情况下， Tomcat Web 管理界面配置为仅限制从 localhost 访问 Manager 和 Host Manager 应用程序。

如果您希望能够从远程 IP 访问 Web 界面，则必须删除这些限制。这可能具有各种安全隐患，不建议用于生产系统。

要从任何地方启用 Web 界面，请打开以下两个文件并注释以下行。

对于 Manager 应用程序，请打开以下文件：

sudo nano /opt/tomcat/latest/webapps/manager/META-INF/context.xml

对于 Host Manager 应用程序，请打开以下文件：

sudo nano /opt/tomcat/latest/webapps/host-manager/META-INF/context.xml
<Context antiResourceLocking="false" privileged="true" >
<!--
  <Valve className="org.apache.catalina.valves.RemoteAddrValve"
         allow="127\.\d+\.\d+\.\d+|::1|0000:1" />
-->
</Context>

另一种选择是仅允许从特定 IP 访问 Manager 和 Host Manager 应用程序。您可以直接将 IP 地址添加到列表中，而不是对块进行注释。

例如，如果您的公共 IP 是 45.45.45.45 您将进行以下更改：

的context.xml

<Context antiResourceLocking="false" privileged="true" >
  <Valve className="org.apache.catalina.valves.RemoteAddrValve"
         allow="127\.\d+\.\d+\.\d+|::1|0000:1|45.45.45.45" />
</Context>

允许的 IP 地址列表是用竖线 | 分隔的列表。您可以添加单个 IP 地址或使用正则表达式。

每次编辑 Tomcat 配置文件以使更改生效时，请记住重新启动 Tomcat 服务：

sudo systemctl restart tomcat

第 7 步：测试 Tomcat 安装

打开浏览器并输入：  http://<your_domain_or_IP_address>:8080

假设安装成功，将出现类似于以下内容的屏幕：

Tomcat Web 应用程序管理器仪表板可在以下位置获得 http://<your_domain_or_IP_address>:8080/manager/html 。从这里，您可以部署，取消部署，启动，停止和重新加载您的应用程序。

您可以使用您在 第 6 步 创建的用户登录。

Tomcat 虚拟主机管理器仪表板可在以下位置获得 http://<your_domain_or_IP_address>:8080/host-manager/html 。从这里，您可以创建，删除和管理 Tomcat 虚拟主机。

结论

您已在 Ubuntu 18.04 系统上成功安装了 Tomcat 9 。您现在可以访问 Apache Tomcat 9 官方文档并了解有关 Apache Tomcat 功能的更多信息。
