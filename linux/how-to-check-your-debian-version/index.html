如何检查Debian Linux版本

May 9, 2019

当您第一次登录Debian Linux系统时，在进行任何工作之前，最好检查一下该机器上运行的Debian版本。

Debian的三个版本总是得到积极维护：

  • 稳定 - 最新正式发布的Debian发行版。在撰写本文时，Debian目前的稳定版本是版本9（延伸）。这是推荐用于生产环境的版本。
  • 测试 - 将成为下一个稳定版本的预览版本。它包含尚未准备好进行稳定版本的软件包，但是它们在队列中。此版本不断更新，直到它被冻结并释放为稳定版本。
  • 不稳定，总是代号为sid - 这是Debian正在积极发展的分布。

在本教程中，我们将展示如何检查系统上安装的Debian Linux版本的几个不同命令。

从命令行检查Debian版本

该lsb_release实用程序显示有关Linux发行版的LSB（Linux标准库）信息。

检查Debian版本的首选方法是使用lsb_release显示有关Linux发行版的LSB（Linux标准库）信息的实用程序。无论您运行的是哪种桌面环境或Debian版本，此方法都可以使用。

lsb_release -a
No LSB modules are available.
Distributor ID:	Debian
Description:	Debian GNU/Linux 9.5 (stretch)
Release:	9.5
Codename:	stretch

您的Debian版本将显示在Description行中。正如您从上面的输出中看到的，我使用的是Debian GNU / Linux 9.5（stretch）。

您可以显示说明行，而不是打印上述所有信息，该行显示通过-d交换机的Debian版本。

lsb_release -d

输出应类似于以下内容：

Description:	Debian GNU/Linux 9.5 (stretch)

或者，您也可以使用以下命令检查Debian版本。

使用该/etc/issue文件检查Debian版本

以下cat命令将显示/etc/issue包含系统标识文本的内容：

cat /etc/issue

输出结果如下所示：

Debian GNU/Linux 9 \n \l

使用该/etc/os-release文件检查Debian版本

/etc/os-release 是一个包含操作系统标识数据的文件，只能在运行systemd的较新的Debian发行版中找到。

只有在Debian 9或更新版本的情况下，此方法才有效：

cat /etc/os-release

输出结果如下所示：

PRETTY_NAME="Debian GNU/Linux 9 (stretch)"
NAME="Debian GNU/Linux"
VERSION_ID="9"
VERSION="9 (stretch)"
ID=debian
HOME_URL="https://www.debian.org/"
SUPPORT_URL="https://www.debian.org/support"
BUG_REPORT_URL="https://bugs.debian.org/"

使用hostnamectl命令检查Debian版本

hostnamectl 是一个允许您设置主机名的命令，但您也可以使用它来检查您的Debian版本。

此命令仅适用于Debian 9或更新版本：

hostnamectl
   Static hostname: debian9.localdomain
         Icon name: computer-vm
           Chassis: vm
        Machine ID: a92099e30f704d559adb18ebc12ddac4
           Boot ID: 4224ba0d5fc7489e95d0bbc7ffdaf709
    Virtualization: qemu
  Operating System: Debian GNU/Linux 9 (stretch)
            Kernel: Linux 4.9.0-8-amd64
      Architecture: x86-64

结论

在本指南中，我们向您展示了如何查找系统上安装的Debian版本。有关Debian发行版的更多信息，请访问Debian发行页面。
