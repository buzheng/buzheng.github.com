如何在 CentOS 7 上安装 MongoDB

April 19, 2019

MongoDB 是一个免费的开源文档数据库。它与 MySQL 和 PostgreSQL 等传统的基于表的 SQL 数据库不同， 被归类为 NoSQL 数据库。

在 MongoDB 中，数据存储在类似 JSON 的灵活文档中，其中字段可能因文档而异。它不需要预定义的模式，数据结构可以随时间更改。

在本教程中，我们将向您展示如何从官方 MongoDB 存储库在 CentOS 7 服务器上安装和配置 MongoDB Community Edition 。

先决条件

在继续本教程之前，请确保以具有 sudo 权限的用户身份登录。

安装 MongoDB

在撰写本文时，官方 MongoDB 存储库中提供的最新版 MongoDB 是 4.0 版。在继续下一步之前，请访问 MongoDB 文档中的 Install on Red Hat 部分，并检查是否有新版本可用。

按照以下步骤在 CentOS 服务器上安装最新的稳定版 MongoDB ：

  1. 启用 MongoDB 存储库

    要将 MongoDB 存储库添加到系统，请打开文本编辑器并在 /etc/yum.repos.d/ 目录中创建一个新的 YUM 存储库配置文件 mongodb-org.repo ：

    文件: /etc/yum.repos.d/mongodb-org.repo

    [mongodb-org-4.0]
name=MongoDB Repository
baseurl=https://repo.mongodb.org/yum/redhat/$releasever/mongodb-org/4.0/x86_64/
gpgcheck=1
enabled=1
gpgkey=https://www.mongodb.org/static/pgp/server-4.0.asc

    如果要安装较旧版本的 MongoDB ，请将每个实例的 “4.0” 替换为您的首选版本。

    1. 安装 MongoDB

    现在已启用存储库，您可以使用 yum 实用程序安装数据包 mongodb-org ：

    sudo yum install mongodb-org

    在安装过程中， yum 将提示您导入 MongoDB GPG 密钥。输入 y 并点击 Enter 。

    以下软件包将作为 mongodb-org 软件包的部分被安装在您的系统上：

    • mongodb-org-server- mongod 守护程序，以及相应的 init 脚本和配置。
    • mongodb-org-mongos- mongos 守护进程。
    • mongodb-org-shell - mongo shell，一个 MongoDB 的交互式 JavaScript 接口，用于执行命令行的管理任务。
    • mongodb-org-tools - 包含几个 MongoDB 工具，用于导入和导出数据，统计信息以及其他实用程序。

  2. 启动 MongoDB

    安装完成后，启动 MongoDB 守护程序并通过键入以下内容启用随机自启动：

    sudo systemctl start mongod
sudo systemctl enable mongod
    1. 验证 MongoDB 安装

    要验证安装，我们将使用该 mongo 工具连接到 MongoDB 数据库服务器并打印服务器版本：

    mongod

    进入 MongoDB shell 后，键入以下命令，该命令将显示 MongoDB 版本：

    db.version()

    输出将如下所示：

    4.0.1

配置 MongoDB

您可以通过编辑以 YAML 编写的配置文件 /etc/mongod.conf 来配置 MongoDB 实例。

在大多数情况下，默认配置设置已足够。但是，对于生产环境，我们建议取消注释安全部分并启用授权，如下所示：

文件： /etc/mongod.conf

security:
  authorization: enabled

这个选项 authorization 启用基于角色的访问控制（RBAC），以控制用户对数据库资源和操作的访问。如果禁用此选项，则每个用户都可以访问任何数据库，并且可以执行任何操作。

更改 MongoDB 配置文件后，重新启动 mongod 服务：

sudo systemctl restart mongod

要查找有关 MongoDB 4.0 中可用配置选项的更多信息，请访问配置文件选项文档页面。

创建管理 MongoDB 用户

如果启用了 MongoDB 身份验证，请创建一个用于访问和管理 MongoDB 实例的管理 MongoDB 用户。

首先访问 mongo shell ：

mongo

进入 MongoDB shell 后，键入以下命令以连接到 admin 数据库：

use admin

switched to db admin

创建一个 mongoAdmin 以该 userAdminAnyDatabase 角色命名的新用户：

db.createUser(
  {
    user: "mongoAdmin", 
    pwd: "changeMe", 
    roles: [ { role: "userAdminAnyDatabase", db: "admin" } ]
  }
)

Successfully added user: {
	"user" : "mongoAdmin",
	"roles" : [
		{
			"role" : "userAdminAnyDatabase",
			"db" : "admin"
		}
	]
}

您可以根据需要为管理 MongoDB 用户命名。

退出 mongo shell ：

quit()

要测试更改，请使用先前创建的管理用户访问 mongo shell ：

mongo -u mongoAdmin -p --authenticationDatabase admin

use admin

switched to db admin

现在，打印用户：

show users

{
	"_id" : "admin.mongoAdmin",
	"user" : "mongoAdmin",
	"db" : "admin",
	"roles" : [
		{
			"role" : "userAdminAnyDatabase",
			"db" : "admin"
		}
	],
	"mechanisms" : [
		"SCRAM-SHA-1",
		"SCRAM-SHA-256"
	]
}

您也可以尝试访问没有任何参数的 mongo shell （只需键入 mongo ），看看是否可以使用与上面相同的命令列出用户。

结论

您已经学习了如何在 CentOS 7 服务器上安装和配置 MongoDB 4.0 。

有关此主题的更多信息，请参阅 The MongoDB 4.0 手册。
