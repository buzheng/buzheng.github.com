如何在Debian 9 Linux上安装Django

May 9, 2019

Django是最流行的Python Web框架，旨在帮助开发人员构建安全，可伸缩和可维护的Web应用程序。

可以使用pip在系统范围内或在Python虚拟环境中安装Django。Django软件包包含在官方Debian存储库中，可以使用apt软件包管理器进行安装。这是在Debian 9上安装Django的最简单方法，但不如在虚拟环境中安装那么灵活。此外，存储库中包含的版本总是在最新版本的Django后面的几个版本。

Python虚拟环境的主要目标是为不同的Python项目创建一个隔离的环境。这样，您可以在一台计算机上拥有多个不同的Django环境，并在每个项目的基础上安装特定版本的模块，而不必担心它会影响您的其他Django安装。如果您将Django安装到全局环境中，那么您只能在计算机上安装一个Django版本。

在Debian 9上安装Django

执行以下步骤在Debian 9上的Python虚拟环境中安装Django。

1.安装Python 3和venv

Debian 9默认提供Python 3.5。您可以通过键入以下内容来验证系统上是否安装了Python 3：

python3 -V

输出应如下所示：

Python 3.5.3

创建虚拟环境的推荐方法是使用该venv模块。

该venv模块包含在python3-venv包中。通过键入以下命令安装它：

sudo apt install python3-venv

安装模块后，我们就可以为Django应用程序创建虚拟环境了。

2.创建虚拟环境

首先导航到要存储Python 3虚拟环境的目录。它可以是您的主目录或您的用户具有读写权限的任何其他目录。

[](/linux/how-to-create-directories-in-linux-with-the-mkdir-command/)为Django应用程序创建一个新目录并导航到它：

mkdir my_django_appcd my_django_app

从目录内部，执行以下命令以创建新的虚拟环境：

python3 -m venv venv

上面的命令创建了一个名为的目录venv，其中包含Python二进制文件的副本，Pip包管理器，标准Python库和其他支持文件。您可以为虚拟环境使用任何名称。

要开始使用虚拟环境，请通过运行activate脚本将其激活：

source venv/bin/activate

激活后，虚拟环境的bin目录将添加到$PATH变量的开头。此外，您的shell提示符也会更改，它将显示您当前使用的虚拟环境的名称。在我们的情况下venv。

3.安装Django

现在虚拟环境处于活动状态，使用Python包管理器安装Django pip：

pip install django

在虚拟环境中，你可以使用命令pip来代替pip3和python代替的python3。

使用以下命令验证安装，该命令将打印Django版本：

python -m django --version

在撰写本文时，最新的Django版本是2.1.2。

2.1.3

您的Django版本可能与此处显示的版本不同。

4.创建Django项目

使用django-admin命令行实用程序创建一个名为的新Django项目mydjangoapp：

django-admin startproject mydjangoapp

上面的命令将mydjangoapp在当前目录中创建一个目录。

tree  mydjangoapp/
mydjangoapp/
|-- manage.py
`-- mydjangoapp
    |-- __init__.py
    |-- settings.py
    |-- urls.py
    `-- wsgi.py

在该目录中，您将找到用于管理已命名项目的主脚本manage.py和另一个目录，包括数据库配置，Django和特定于应用程序的设置。

让我们迁移数据库并创建一个管理用户。

首先，导航到mydjangoapp目录：

cd mydjangoapp

SQLite是Django的默认数据库。对于生产应用程序，您可以使用PostgreSQL，MariaDB，Oracle或MySQL数据库。

键入以下命令迁移数据库：

python manage.py migrate

输出将如下所示：

python manage.py createsuperuser

该命令将提示您输入用户名，电子邮件地址和密码。

5.测试开发服务器

使用manage.py脚本后跟runserver选项启动开发Web服务器：

python manage.py runserver

您将看到以下输出：

Performing system checks...

System check identified no issues (0 silenced).
October 20, 2018 - 11:16:28
Django version 2.1.2, using settings 'mydjangoapp.settings'
Starting development server at http://127.0.0.1:8000/
Quit the server with CONTROL-C.

如果您在虚拟机上安装了Django并且想要访问Django开发服务器，那么您需要编辑该settings.py文件并在ALLOWED_HOSTS列表中添加服务器IP地址。

http://127.0.0.1:8000在您的Web浏览器中打开，您将看到默认的Django登录页面：

要访问Django管理界面，请添加/admin到URL（http://127.0.0.1:8000/admin/）的末尾。这将带您进入管理员登录界面：

输入您的用户名和密码，您将被重定向到Django管理页面：

停止CTRL-C终端中的开发服务器类型。

6.停用虚拟环境

完成您的工作后，通过键入来停用环境，deactivate您将返回到正常的shell。

deactivate

结论

您已经学习了如何在Debian 9系统上创建Python虚拟环境并安装Django。要创建其他Django开发环境，请重复本教程中概述的步骤。

如果您是Django的新手，请访问Django文档页面，了解如何开发您的第一个Django应用程序。
