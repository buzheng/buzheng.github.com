如何在 CentOS 7 上安装 exFAT 驱动器

July 15, 2019

exFAT( 扩展文件分配表)是一种专有的 Microsoft 文件系统，针对闪存设备(如 SD 卡和 USB 闪存驱动器)进行了优化。它旨在取代旧的 32 位 FAT32 文件系统，FAT32 文件系统无法存储大于 4 GB 的文件。

最新版本的 Windows 和 macOS 操作系统支持 exFAT 文件系统。与大多数其他主要 Linux 发行版一样， CentOS 默认情况下不支持专有的 exFAT 文件系统。

如果您使用 CentOS 作为操作系统，那么在尝试安装 exFAT 格式的 USB 驱动器时，您可能会遇到问题。

本教程介绍如何在 CentOS 7 上启用 exFAT 支持。

在 CentOS 上安装 exFAT Drive

为了能够在 CentOS 上安装 exFAT 文件系统，您需要安装免费的 FUSE exFAT 模块和工具，它为类 Unix 系统提供全功能的 exFAT 文件系统实现。

在 CentOS 7 核心存储库中不提供 exFAT 包。您可以选择从源构建 exFAT 工具，或者使用 Nux Dextop 存储库中的 yum 进行安装。我们将选择第二种方案。

Nux 存储库依赖于 EPEL 软件存储库。如果系统上未启用 EPEL 存储库，请键入以下命令启用它：

sudo yum install epel-release

接下来，导入存储库 GPG 密钥并通过安装 rpm 软件包启用 Nux 存储库：

sudo rpm -v --import http://li.nux.ro/download/nux/RPM-GPG-KEY-nux.ro
sudo rpm -Uvh http://li.nux.ro/download/nux/dextop/el7/x86_64/nux-dextop-release-0-5.el7.nux.noarch.rpm

启用存储库后 exfat-fuse ，使用以下命令安装 exfat-utils 包：

sudo yum install exfat-utils fuse-exfat

您现在可以安装使用 exFAT 格式的设备了。

结论

我们已经向您展示了如何在 CentOS 7 计算机上启用对 exFAT 文件系统的支持。有些人将 exFAT 称为 FAT64 。

插入时 USB 驱动器将自动安装，但如果自动安装失败，则必须手动安装驱动器。
