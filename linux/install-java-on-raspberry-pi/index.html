树莓派 Raspberry Pi 安装 Java 教程

2020年1月17日 • 约 1100 字 , 预计阅读时间 5 分钟

Java 是用于构建各种应用程序和系统的最流行的编程语言之一。

本指南说明了如何在运行最新 Raspbian 操作系统的 Raspberry Pi 上安装 Java(OpenJDK) 。

Java 有两种不同的实现， Oracle Java 和 OpenJDK 。 OpenJDK 是 Java 平台的开源实现。 Oracle Java 具有一些其他的商业功能，并且其许可证仅允许非商业用途，例如个人或开发用途。

标准的 Raspbian 存储库包括两个不同的 Java 软件包： Java 运行时环境 (JRE) 和 Java 开发工具包 (JDK) 。 JRE 包括 Java 虚拟机 (JVM) ，类和二进制文件，使您可以运行 Java 程序。 JDK 包含 JRE 以及构建 Java 应用程序所需的开发/调试工具和库。

如果不确定要安装哪个 Java 软件包，通常建议坚持使用默认的 OpenJDK(JDK 11) 版本。某些基于 Java 的应用程序可能需要特定版本的 Java ，因此您应查阅应用程序文档。

在 Raspberry Pi 上安装 Java 11

OpenJDK 11 是最新的 Raspbian 操作系统中的默认 Java 开发和运行时，该操作系统基于 Debian 10 Buster 。

运行以下命令在您的 Raspberry Pi 上安装 OpenJDK 11 JDK ：

sudo apt update

安装完成后，通过检查 Java 版本进行验证：

java -version

输出应如下所示：

openjdk version "11.0.5" 2019-10-15
OpenJDK Runtime Environment (build 11.0.5+10-post-Raspbian-1deb10u1)
OpenJDK Server VM (build 11.0.5+10-post-Raspbian-1deb10u1, mixed mode)

现在，您已经在 Pi 上成功安装了 Java ，然后就可以开始使用它了。

在 Raspberry Pi 上安装 Java 8

以前的 Java LTS 版本 8 仍受支持并广泛使用。如果您的应用程序需要 Java 8 ，请输入以下内容进行安装：

sudo apt update
sudo apt install openjdk-8-jdk openjdk-8-jre

通过打印 Java 版本来验证安装：

java -version

输出应如下所示：

openjdk version "1.8.0_212"
OpenJDK Runtime Environment (build 1.8.0_212-8u212-b01-1+rpi1-b01)
OpenJDK Client VM (build 25.212-b01, mixed mode)

设置默认版本

如果您的 Pi 上安装了多个 Java 版本，请运行以下 java -version 命令以检查默认版本：

java -version

要更改默认版本，请使用该 update-alternatives 工具：

sudo update-alternatives --config java

输出将如下所示：

There are 2 choices for the alternative java (providing /usr/bin/java).

  Selection    Path                                            Priority   Status
------------------------------------------------------------
* 0            /usr/lib/jvm/java-11-openjdk-armhf/bin/java      1111      auto mode
  1            /usr/lib/jvm/java-11-openjdk-armhf/bin/java      1111      manual mode
  2            /usr/lib/jvm/java-8-openjdk-armhf/jre/bin/java   1081      manual mode

Press <enter> to keep the current choice[*], or type selection number:

您将看到所有已安装 Java 版本的列表。输入要设置为默认版本的版本号，然后按 Enter。

JAVA_HOME 环境变量

JAVA_HOME 环境变量用来确定 Java 安装位置，他会被一些Java应用程序使用。

要设置 JAVA_HOME 环境变量，请使用以下 update-alternatives 命令查找 Java 的安装位置：

sudo update-alternatives --config java

在此示例中，安装路径如下：

  • OpenJDK 11位于 /usr/lib/jvm/java-11-openjdk-armhf/bin/java
  • OpenJDK 8位于 /usr/lib/jvm/java-8-openjdk-armhf/jre/bin/java

找到 Java 安装路径后，打开 /etc/environment 文件：

sudo nano /etc/environment

假设要设置 JAVA_HOME 为 OpenJDK 11，在文件末尾添加以下行：

/ etc /环境

JAVA_HOME="/usr/lib/jvm/java-11-openjdk-armhf/bin/java"

为了使更改在当前的 shell 程序上生效，您可以重新打开 设立了 或运行以下source命令：

source /etc/environment

要验证是否 JAVA_HOME 设置了变量，请键入：

echo $JAVA_HOME

您应该看到 Java 11 二进制文件的路径：

/usr/lib/jvm/java-11-openjdk-armhf/bin/java

/etc/environment 是系统范围的配置文件，供所有用户使用。如果要基于每个用户设置 JAVA_HOME 变量，请将该行添加到 .bashrc 用户登录时加载的或任何其他配置文件中。

卸载Java

您可以使用 apt 直接卸载Java，就像与随系统安装的任何其他软件包一样。

例如，要卸载 default-jdk 软件包，只需运行：

sudo apt remove default-jdk

结论

默认的 Raspbian 存储库中提供了 OpenJDK 的最新 LTS 版本，安装是一个简单明了的任务。
