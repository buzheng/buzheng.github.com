如何在 CentOS 7 上安装 Apache

March 11, 2019

Apache HTTP 服务器是世界上最受欢迎的 Web 服务器。它是一个免费的，开源的跨平台 HTTP 服务器，可以通过大量的扩展模块来提供强大的功能。下面的部分介绍了如何在 CentOS 7 服务器上安装和管理 Apache Web 服务器。

先决条件

在开始本教程之前，请确保以具有 sudo 权限的用户身份登录。

安装 Apache

Apache 可以在 CentOS 的自有软件仓库中使用，安装非常简单。

在 CentOS 和 RHEL 上 Apache 包和服务的名称是 httpd。请运行以下命令安装程序包：

sudo yum install httpd

安装完成后，启用随机启动并启动 Apache 服务：

sudo systemctl enable httpd
sudo systemctl start httpd

调整防火墙

如果您的服务器受防火墙保护，则需要打开 HTTP 80 端口 和 HTTPS 443 端口。使用以下命令打开必要的端口：

sudo firewall-cmd --permanent --zone=public --add-service=http
sudo firewall-cmd --permanent --zone=public --add-service=https
sudo firewall-cmd --reload

验证 Apache 安装

现在我们已经在 CentOS 7 服务器上安装并运行了 Apache ，我们可以通过以下命令来查看 Apache 服务的状态和版本。

管理 Apache 服务

您可以像管理任何其他 systemd 单元一样管理 Apache 服务。

要停止 Apache 服务，请运行：

sudo systemctl stop httpd

要再次启动它，请键入：

sudo systemctl start httpd

要重新启动 Apache 服务：

sudo systemctl restart httpd

在进行一些配置更改后重新加载 Apache 服务：

sudo systemctl reload httpd

如果要禁用 Apache 服务随机自启动：

sudo systemctl disable httpd

并重新启用它：

sudo systemctl enable httpd

Apache 配置文件的结构和最佳实践

  • 所有 Apache 配置文件都位于 /etc/httpd 目录中。
  • 主要的 Apache 配置文件是 /etc/httpd/conf/httpd.conf 。
  • 所有位于 /etc/httpd/conf.d 目录中以 .conf 结尾的配置文件都会包含进主 Apache 配置文件中。
  • 负责加载各种 Apache 模块的配置文件位于 /etc/httpd/conf.modules.d 目录中。
  • 为了更好的可维护性，建议为每个域创建单独的配置文件 (vhost)。
  • 新的 Apache vhost 文件必须 .conf 结尾并保存在 /etc/httpd/conf.d 目录中。您可以根据需要拥有尽可能多的虚拟主机。
  • 遵循标准命名约定是个好主意，例如，如果您的域名是， mydomain.com 则应该命名配置文件 /etc/httpd/conf.d/mydomain.com.conf
  • Apache 日志文件 (access_log 和 error_log)位于 /var/log/httpd/ 目录中。建议每个虚拟主机使用不同 access 和 error 日志文件。
  • 您可以将网站根目录设置为任何位置。 webroot 最常见的位置包括：
    • /home/<user_name>/<site_name>
    • /var/www/<site_name>
    • /var/www/html/<site_name>
    • /opt/<site_name>

结论

您已在 CentOS 7 服务器上成功安装了 Apache 。您现在已准备好开始部署应用程序并将 Apache 用作 Web 或代理服务器。
