如何在 Ubuntu 18.04 上安装 Jenkins

May 5, 2019

Jenkins 是一个开源自动化服务器，提供了一种简单的方法来达成持续集成和连续交付(CI/CD)。

持续集成(CI)是一种 DevOps 实践，其中团队成员定期将其代码更改提交到版本控制软件仓库，然后运行自动构建和测试。持续交付(CD)是一系列实践，其中代码更改自动构建，测试并部署到生产中。

在本教程中，我们将向您展示如何使用 Jenkins 的 Debian 软件仓库在 Ubuntu 18.04 计算机上安装 Jenkins 。

虽然本教程是为 Ubuntu 18.04 Bionic Beaver 编写的，但 Ubuntu 16.04 Xenial Xerus 可以使用相同的步骤。

先决条件

在继续本教程之前，请确保以具有 sudo 权限的用户身份登录。

安装 Jenkins

要在您的 Ubuntu 系统上安装 Jenkins ，请按照下列步骤操作：

  1. 安装 Java

    由于 Jenkins 是一个 Java 应用程序，因此第一步是安装 Java 。使用以下命令更新程序包索引并安装 Java 8 OpenJDK 程序包：

    sudo apt update
sudo apt install openjdk-8-jdk

    当前版本的 Jenkins 尚不支持Java 10 和 Java 11。如果您的计算机上安装了多个 Java 版本，请确保 Java 8 是默认的 Java 版本。

    1. 添加 Jenkins Debian 软件仓库

    使用以下 wget 命令导入 Jenkins 软件仓库的 GPG 密钥：

    wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -

    上面的命令应该输出 OK ，这意味着密钥已成功导入，并且来自此软件仓库的包将被视为可信。

    接下来，将 Jenkins 软件仓库添加到系统中：

    sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'
    1. 安装 Jenkins

    启用 Jenkins 软件仓库后，通过键入以下 apt 命令更新软件包列表并安装最新版本的 Jenkins ：

    sudo apt update
sudo apt install jenkins

    安装过程完成后， Jenkins 服务将自动启动。您可以通过打印服务状态来验证它：

    systemctl status jenkins

    你应该看到类似的东西：

    ● jenkins.service - LSB: Start Jenkins at boot time
Loaded: loaded (/etc/init.d/jenkins; generated)
Active: active (exited) since Wed 2018-08-22 13:03:08 PDT; 2min 16s ago
    Docs: man:systemd-sysv-generator(8)
    Tasks: 0 (limit: 2319)
CGroup: /system.slice/jenkins.service

调整防火墙

如果要在受防火墙保护的远程 Ubuntu 服务器上安装 Jenkins ，则需要打开端口 8080 。假设您正在使用 UFW 管理防火墙，可以使用以下命令打开端口：

sudo ufw allow 8080

验证更改：

sudo ufw status
Status: active

To                         Action      From
--                         ------      ----
OpenSSH                    ALLOW       Anywhere
8080                       ALLOW       Anywhere
OpenSSH (v6)               ALLOW       Anywhere (v6)
8080 (v6)                  ALLOW       Anywhere (v6)

设置 Jenkins

要设置刚刚安装的 Jenkins ，请打开浏览器，键入您的域或 IP 地址，然后键入端口 8080(http://your_ip_or_domain:8080)，将显示类似以下内容的屏幕：

在安装过程中， Jenkins 安装程序会创建一个 32 个字符的长字母数字密码。使用以下命令在终端上打印密码：

sudo cat /var/lib/jenkins/secrets/initialAdminPassword
2115173b548f4e99a203ee99a8732a32

从终端复制密码，将其粘贴到管理员密码字段中，然后单击 Continue 。

在下一个屏幕上，安装向导将询问您是要安装建议的插件还是要选择特定的插件。单击该 Install suggested plugins 框，安装过程将立即开始。

安装插件后，系统将提示您设置第一个管理员用户。填写所有必填信息，然后单击 Save and Continue 。

下一页将要求您设置 Jenkins 实例的 URL 。该字段将填充自动生成的 URL 。

单击 Save and Finish 按钮确认 URL ，完成设置过程。

单击该 Start using Jenkins 按钮，您将被重定向到以前面一个步骤中创建的管理员用户身份登录的 Jenkins 仪表板。

此时，您已在系统上成功安装了 Jenkins 。

结论

在本教程中，您学习了如何安装和执行 Jenkins 的初始配置。您现在可以通过访问官方 Jenkins 文档页面来开始探索 Jenkins 功能。
