在 CentOS 8 上安装和使用 curl 命令

2020年2月14日 • 约 500 字 , 预计阅读时间 3 分钟

命令行工具 curl 的主要功能是用于从远程服务器或向远程服务器传输数据。使用 curl ，您可以使用各种网络协议(例如 HTTP ， HTTPS ， SCP ， SFTP 和 FTP) 下载或上传数据。

如果您在尝试使用 curl 来下载文件时收到这样一条错误消息: curl command not found ，表示您的 CentOS 计算机上未安装 curl 软件包。

本文提供有关如何在 CentOS 8 上安装和使用 curl 命令的说明。

在 CentOS 上安装 curl

标准 CentOS 8 储存库中提供了 Curl 软件包。要安装它，请运行以下命令：

sudo dnf install curl

安装完成后，请在终端中输入 curl 进行验证：

curl

输出应如下所示：

curl: try 'curl --help' or 'curl --manual' for more information

就这样， curl 已安装在您的 CentOS 系统上，您可以开始使用它。

使用  curl

不带任何选项使用时， curl 将给定 URL 的源代码打印到标准输出中：

curl https://example.com

要使用 curl 下载文件，请使用 -o 或 -O 选项。

小写 -o 选项允许您指定保存文件的名称：

curl -o linux.tar.xz https://cdn.kernel.org/pub/linux/kernel/v5.x/linux-5.5.3.tar.xz

大写 -O 以其原始文件名保存文件：

curl -O https://cdn.kernel.org/pub/linux/kernel/v5.x/linux-5.5.3.tar.xz

与 -I 选项一起使用时，将 curl 显示给定 URL 的 HTTP 标头：

curl -I https://www.centos.org/

HTTP/1.1 200 OK
Date: Thu, 13 Feb 2020 22:01:04 GMT
Server: Apache/2.4.6 (CentOS) OpenSSL/1.0.2k-fips
Strict-Transport-Security: max-age=31536000
X-Frame-Options: SAMEORIGIN
X-Xss-Protection: 1; mode=block
X-Content-Type-Options: nosniff
Referrer-Policy: same-origin
Last-Modified: Thu, 06 Feb 2020 17:21:08 GMT
ETag: "5421-59deb7fadfdfd"
Accept-Ranges: bytes
Content-Length: 21537
Content-Type: text/html; charset=UTF-8

使用 curl 您还可以从 FTP 服务器下载受密码保护的文件：

curl -u FTP_USERNAME:FTP_PASSWORD ftp://ftp.example.com/file.tar.gz

结论

curl 是一种多功能工具，可让您通过网络发送和接收数据。

有关如何使用此工具的更多信息，请参见 “curl 命令示例” 。
