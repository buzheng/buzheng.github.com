如何在 Debian 9 上安装Git

April 23, 2019

本教程将向您展示如何在 Debian 9 上安装和配置 Git 。

Git 是许多开源和商业项目使用的世界上最流行的分布式版本控制系统。它允许您跟踪代码更改，恢复到以前的阶段，创建分支以及与您的开发人员协作。

Git 最初是由 Linux 内核的创建者 Linus Torvalds 开发的。

本教程在 Debian 9 上进行了测试，但它也适用于任何以前的 Debian 版本。

先决条件

在继续本教程之前，请确保以具有 sudo 权限的用户身份登录。

用 apt 安装 Git

安装 Git 的最简单和推荐的方法是使用 Debian 默认存储库中的 apt 包管理工具。如果要安装最新的稳定版 Git ，请转到本教程 “从源代码安装 Git” 部分。

以下步骤将向您展示如何在 Debian 系统上安装 Git ：

  1. 更新包索引。

    在安装新软件包之前，您应该始终更新 apt 软件包索引：

    sudo apt update
    1. 安装 Git 。

    更新列表后，请发出以下命令来安装 Git ：

    sudo apt install git

  2. 验证 Git 安装。

    要验证安装类型，请使用以下命令打印 Git 版本：

    git --version
    git version 2.11.0

从上面的输出中可以看出，您已经成功安装了 Git 版本 2.11.0 。您现在可以转到本教程的 “配置 Git” 部分来完成设置。

从源代码安装 Git

另一个安装方法是从源代码编译 Git ，您将会安装最新的 Git 版本并自定义构建选项，但您将无法通过 apt 包管理器维护您的 Git 安装。

在继续后续步骤之前，首先需要安装在 Debian 系统上构建 Git 所需的软件包：

sudo apt update
sudo apt install make libssl-dev libghc-zlib-dev libcurl4-gnutls-dev libexpat1-dev gettext unzip

一旦安装了依赖项，打开浏览器，转到 GitHub 上的 Git 项目镜像，并复制以 .tar.gz 结尾的最新版本链接地址：

在撰写本文时，最新的稳定 Git 版本是 2.18.0 。

我们将在 /usr/src 目录中下载 Git 源代码，该目录是放置源文件的常用位置，首先跳转到目录：

cd /usr/src/

使用 wget 命令将存档文件下载为 git.tar.gz ：

sudo wget https://github.com/git/git/archive/v2.18.0.tar.gz -O git.tar.gz

下载完成后，解压缩下载的文件，然后输入以下命令切换到 git 源目录：

sudo tar -xf git.tar.gzcd git-*

现在，您可以通过键入以下两个命令来编译和安装 Git ：

sudo make prefix=/usr/local all
sudo make prefix=/usr/local install

安装完成后，通过键入以下命令验证它，该命令将打印已安装的 Git 版本：

git --version
git version 2.18.0

如果以后要升级到较新版本，则需要重复上面的安装过程。

配置 Git

现在你已经安装了 Git ，建议你设置你的用于 Git 提交的电子邮件和用户名：

git config --global user.name "Your Name"
git config --global user.email "youremail@yourdomain.com"

您可以使用以下命令验证更改：

git config --list
user.name=Your Name
user.email=youremail@yourdomain.com

配置设置存储在 ~/.gitconfig 文件中：

[user]
    name = Your Name
    email = youremail@yourdomain.com

如果要对 Git 配置进行其他更改，可以使用该 git config 命令或 ~/.gitconfig 手动编辑文件。

结论

您已经学习了如何在 Debian 系统上安装 Git 。您现在应该查看在线书籍 Pro Git并了解有关如何使用 Git 的更多信息。
