如何在 Ubuntu 18.04 上创建 Python 虚拟环境

June 9, 2019

Python 虚拟环境是一个独立的目录树，包括 Python 安装和许多其他包。

Python 虚拟环境的主要目的是为不同的 Python 项目创建一个独立的环境。这样，您可以在每个项目的基础上安装特定版本的模块，而不必担心它会影响您的其他 Python 项目。

在本教程中，我们将提供有关如何在 Ubuntu 18.04 上创建 Python 虚拟环境的分步说明。

为 Python 3 创建虚拟环境

默认情况下， Ubuntu 18.04 附带 Python 3.6 。您可以通过运行以下命令验证系统上是否安装了 Python 3 ：

python3 -V

输出应如下所示：

Python 3.6.5

如果要安装 Python 语言的最新主要版本: Python 3.7 , 请遵循此说明。

从 Python 3.6 开始，创建虚拟环境的推荐方法是使用该 venv 模块。

我们首先安装 python3-venv 提供 venv 模块的软件包。

sudo apt install python3-venv

安装模块后，我们就可以为 Python 3 创建虚拟环境了。

首先切换到您希望存储 Python 3 虚拟环境的目录。在目录中运行以下命令以创建新的虚拟环境：

python3 -m venv my-project-env

上面的命令创建了一个名为的目录 my-project-env ，其中包含 Python 二进制文件的副本， Pip 包管理器，标准 Python 库和其他支持文件。

要开始使用此虚拟环境，您需要通过运行 activate 脚本来激活它：

source my-project-env/bin/activate

激活后，虚拟环境的 bin 目录将添加到 $PATH 变量的开头。此外，您的 shell 提示符也会更改，它将显示您当前使用的虚拟环境的名称。在我们的情况下是 my-project-env ：

$ source my-project-env/bin/activate
(my-project-env) $

现在虚拟环境已激活，我们可以使用 pip 开始安装，升级和删除软件包。

让我们使用 Requests 模块创建一个简单的 Python 脚本。

在虚拟环境中，你可以使用命令 pip 来代替 pip3 和 python 代替的 python3 。

第一步是使用 Python 包管理器 pip 安装模块：

pip install requests

要验证安装，您可以尝试导入模块：

import requests

如果导入模块没有错误，则安装成功。

在此示例中，我们将使用 httpbin.org 站点，该站点提供简单的 HTTP 请求和响应服务来打印所有标头条目。

打开文本编辑器并创建一个新文件：

nano testing.py

将以下内容粘贴到文件中：

import requests

r = requests.get('http://httpbin.org/get')  
print(r.headers)

关闭并保存文件。

我们现在可以输入以下命令来运行脚本：

python testing.py

该脚本将打印所有标题条目的字典，如下所示：

{'Connection': 'keep-alive', 'Server': 'gunicorn/19.9.0', 'Date': 'Tue, 18 Sep 2018 16:50:03 GMT', 'Content-Type': 'application/json', 'Content-Length': '266', 'Access-Control-Allow-Origin': '*', 'Access-Control-Allow-Credentials': 'true', 'Via': '1.1 vegur'}

完成工作以停用环境后，只需键入即可 deactivate 返回正常的 shell 。

deactivate

结论

您已经学习了如何创建和使用 Python 虚拟环境。您可以重复上面概述的步骤，并为 Python 项目创建其他虚拟环境。
