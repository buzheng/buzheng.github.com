如何在Ubuntu 18.04上禁用防火墙

2019年5月9日 • 约 800 字 , 预计阅读时间 4 分钟

Ubuntu附带了一个名为UFW（Uncomplicated Firewall）的防火墙配置工具。UFW是一个用户友好的前端，用于管理iptables防火墙规则，其主要目标是使管理防火墙规则更容易，或者名称不复杂。

强烈建议启用防火墙。但是，在某些情况下，例如测试，您可能需要停止或禁用防火墙。

在本教程中，我们将向您展示如何在Ubuntu 18.04系统上禁用UFW防火墙。

先决条件

确保以具有sudo权限的用户身份登录。

检查防火墙状态

要查看UFW防火墙的当前状态，请使用以下ufw status命令：

sudo ufw status

默认情况下禁用UFW防火墙。如果您之前从未激活过UFW，输出将如下所示：

Status: inactive

否则，如果启用了防火墙，您将看到以下消息：

Status: active

禁用防火墙

如果由于遇到连接问题或者配置防火墙时遇到问题而禁用防火墙，请查看我们的指南，了解如何在Ubuntu 18.04上使用UFW设置防火墙。

要在Ubuntu系统上禁用UFW防火墙，请使用以下ufw disable命令：

sudo ufw disable

输出将如下所示：

Firewall stopped and disabled on system startup

上面的命令将停止并禁用防火墙，但它不会删除防火墙规则。下次启用防火墙时，将加载相同的规则。

如果要禁用防火墙并删除所有防火墙规则，请使用以下ufw reset命令：

sudo ufw reset

系统将提示您是否要继续操作：

Resetting all rules to installed defaults. This may disrupt existing ssh
connections. Proceed with operation (y|n)? 

Backing up 'user.rules' to '/etc/ufw/user.rules.20190122_115214'
Backing up 'before.rules' to '/etc/ufw/before.rules.20190122_115214'
Backing up 'after.rules' to '/etc/ufw/after.rules.20190122_115214'
Backing up 'user6.rules' to '/etc/ufw/user6.rules.20190122_115214'
Backing up 'before6.rules' to '/etc/ufw/before6.rules.20190122_115214'
Backing up 'after6.rules' to '/etc/ufw/after6.rules.20190122_115214'

当您想要还原所有更改并重新开始时，重置UFW防火墙非常有用。

启用防火墙

在启用防火墙之前，请确保SSH端口已打开以进行连接。

要启用防火墙运行：

sudo ufw enable

当询问类型y继续操作时：

Command may disrupt existing ssh connections. Proceed with operation (y|n)?

Firewall is active and enabled on system startup

结论

在本教程中，您学习了如何在Ubuntu 18.04计算机上停止并永久禁用防火墙。

大多数UFW命令都是直观且易于记忆的。例如，ufw status显示防火墙的状态并ufw disable禁用防火墙。
