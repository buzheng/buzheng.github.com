如何在 CentOS 7 上安装 Python 3

2019年3月17日 • 约 1200 字 , 预计阅读时间 5 分钟

本教程将指导您使用Software Collections (SCL) 在 CentOS 7 系统上安装 Python 3 ，SCL 默认 Python 发行版本 2.7 在一起的。我们还将向您展示如何创建 Python 虚拟环境。

Python 是世界上最流行的编程语言之一。凭借其简单易学的语法， Python 是初学者和经验丰富的开发人员的绝佳选择。 Python 是一种非常通用的编程语言，您可以使用它来完成您想要的任何操作，编写小脚本，构建游戏，开发网站，创建机器学习算法，分析数据等等。

许多流行的应用程序和网站，包括 YouTube ， DropBox ， Reddit ， Quora ， Instagram ， Pinterest 都是使用 Python 开发的。

虽然 Python 2 得到了很好的支持和活跃，但 Python 3 被认为是该语言的现在和未来。

启用 Software Collections (SCL)

Software Collections，也称为 SCL ，是一个社区项目，允许您在同一系统上构建，安装和使用多个版本的软件，而不会影响系统默认软件包。通过启用 Software Collections，您将可以访问核心存储库中不可用的较新版本的编程语言和服务。

CentOS 7 附带 Python 2.7.5 ，这是 CentOS 基础系统的关键部分。 SCL 将允许您在同时安装 python v2.7.5 的情况下安装较新版本的 python 3.x ，以便系统工具等 yum 继续正常工作。

为了启用 SCL ，我们需要安装 CentOS SCL 发行文件。它是 CentOS extras 存储库的一部分，可以通过运行以下命令来安装：

sudo yum install centos-release-scl

在 CentOS 7 上安装 Python 3

现在我们可以访问 SCL 存储库，我们可以安装我们需要的任何 Python 3.x 版本。目前可以使用以下 Python 3 集合：

  • Python 3.3
  • Python 3.4
  • Python 3.5
  • Python 3.6

在本教程中，我们将安装 Python 3.6 ，这是撰写本文时可用的最新版本。为此，请在 CentOS 7 终端上输入以下命令：

sudo yum install rh-python36

使用 Python 3

安装软件包 rh-python36 后，输入以下命令检查 Python 版本：

python --version

Python 2.7.5

您会注意到 Python 2.7 是当前 shell 中的默认 Python 版本。

要访问 Python 3.6 ，您需要使用 Software Collection scl 工具启动新的 shell 实例：

scl enable rh-python36 bash

上面的命令是调用 /opt/rh/rh-python36/enable 更改 shell 环境变量的脚本。

如果再次检查 Python 版本，您会发现 Python 3.6 现在是当前 shell 中的默认版本。

python --version

Python 3.6.3

需要指出的是， Python 3.6 仅在此 shell 会话中设置为默认的 Python 版本。如果退出会话或从另一个终端打开一个新会话， Python 2.7 将是默认的 Python 版本。

安装开发工具

构建 Python 模块需要开发工具，您可以通过输入以下内容来安装必要的工具和库：

sudo yum groupinstall 'Development Tools'

创建虚拟环境

Python Virtual Environments 允许您在特定项目的隔离位置安装 Python 模块，而不是全局安装。这样您就不必担心影响其他 Python 项目。

在 Python 3 中创建新虚拟环境的首选方法是执行 venv 命令。

假设我们想 my_new_project 在我们的用户主目录和匹配的虚拟环境中创建一个新的 Python 3 项目。

首先创建项目目录并切换到它：

mkdir ~/my_new_project
cd ~/my_new_project

使用该 scl 工具激活 Python 3.6 ：

scl enable rh-python36 bash

从项目根目录内部运行以下命令以创建名为 my_project_venv 的虚拟环境 ：

python -m venv my_project_venv

要首先使用虚拟环境，我们需要输入以下命令来激活它：

source my_project_venv/bin/activate

激活环境后， shell 提示符将以环境名称作为前缀：

(my_project_venv) user@host:~/my_new_project$

从 Python 3.4 开始，在创建虚拟环境时，默认情况下会安装 Python 的包管理器 pip 。

结论

您现在应该在 CentOS 7 计算机上安装 Python 3 编程环境，然后就可以开始开发 Python 3 项目了。
