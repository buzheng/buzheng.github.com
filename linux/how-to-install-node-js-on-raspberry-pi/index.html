如何在 Raspberry Pi 上安装 Node.js 和 npm

2019年6月3日 • 约 1100 字 , 预计阅读时间 5 分钟

Node.js 是一个开源的跨平台 JavaScript 运行时环境，构建于 Chrome 的 JavaScript 引擎 V8 之上，允许在服务器端执行 JavaScript 代码。它主要用于构建服务器端应用程序，但它作为全栈和前端解决方案也非常受欢迎。 npm 是 Node.js 的默认包管理器，也是世界上最大的软件注册表。

在本教程中，我们将解释如何在 Raspberry Pi 上安装 Node.js 和 npm 。我们假设您在 Raspberry Pi 上安装了 Raspbian 。

从 NodeSource 存储库安装 Node.js 和 npm

NodeSource 是一家专注于提供企业级 Node 支持的公司。 NodeSource 维护一个包含最新版本 Node.js 的 APT 存储库。

通过在终端中运行以下命令来启用 NodeSource 存储库：

curl -sL https://deb.nodesource.com/setup_10.x | sudo bash -

Node.js 的当前 LTS 版本是版本 10.x (Dubnium)。如果你想安装 Node.js 的 8.x 版本只是改变 setup_10.x, 替换为 setup_8.x

启用存储库后，键入以下命令安装 Node.js 和 npm ：

sudo apt install nodejs

要验证安装，请运行以下命令，该命令将打印 Node.js 版本：

node --version

v10.16.0

而已。您已在 Raspberry Pi 板上成功安装了 Node.js 和 npm 。

使用 NVM 安装 Node.js 和 NPM

NVM( 节点版本管理器)是一个 bash 脚本，允许您安装和管理多个 Node.js 版本。如果需要安装特定的 Node.js 版本，或者需要在 Raspberry Pi 上安装多个 Node.js 版本，请使用此方法。

要安装 nvm ，请运行以下 curl 命令，该命令将下载并运行 nvm 安装脚本：

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.34.0/install.sh | bash

安装脚本将 nvm 存储库从 Github  克隆到 ~/.nvm 目录，并将 nvm 路径添加到 Bash 配置文件。

=> Close and reopen your terminal to start using nvm or run the following to use it now:

export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"  # This loads nvm
[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"  # This loads nvm bash_completion

如输出所示，您可以打开新的 shell 会话，也可以运行那些将添加当前会话路径的命令。做任何更容易的事情。

要确保正确安装 nvm ，请键入：

nvm --version

0.34.00

您现在可以通过运行以下命令安装最新的 Node.js 版本：

nvm install node

验证安装：

node --version

v12.3.1

为了更好地解释 nvm 如何工作，我们将安装另外两个版本，最新的 LTS 版本和版本 8.9.4 。

nvm install --ltsnvm install 8.9.4

安装两个版本后，键入以下内容列出 Node.js 实例：

nvm ls

->       v8.9.4
       v10.16.0
        v12.3.1
default -> node (-> v12.3.1)
node -> stable (-> v12.3.1) (default)
stable -> 12.3 (-> v12.3.1) (default)
iojs -> N/A (default)
unstable -> N/A (default)
lts/* -> lts/dubnium (-> v10.16.0)
lts/argon -> v4.9.1 (-> N/A)
lts/boron -> v6.17.1 (-> N/A)
lts/carbon -> v8.16.0 (-> N/A)
lts/dubnium -> v10.16.0

在上面的输出中，右侧带箭头的条目 (-> v8.9.4) 是当前 shell 会话中使用的版本，默认版本设置为 v12.3.1 。

默认版本是打开新 shell 会话时将使用的版本。

要更改当前活动的版本为 v10.16.0， 使用以下命令：

nvm use 10.16.0

并通过键入以下来验证它：

nvm current

v10.16.0

如果要将版本 10.16.0 设置为默认的 Node.js 版本类型：

nvm alias default 10.16.0

安装开发工具

为了能够从 npm 注册表编译和安装本机加载项，您需要安装开发工具：

sudo apt install build-essential

卸载 Node.js.

如果由于某些原因要卸载 Node.js 包，可以使用以下命令：

sudo apt remove nodejs

结论

我们已经向您展示了两种在 Raspberry Pi 板上安装 Node.js 和 npm 的方法。您选择的方法取决于您的要求和偏好。尽管从 NodeSource 存储库安装打包版本更容易，但 nvm 方法为您提供了更大的灵活性，可以在每个用户的基础上添加和删除不同的 Node.js 版本。

现在您已经在 Raspberry Pi 系统上安装了 Node.js ，您可以开始开发应用程序了。
